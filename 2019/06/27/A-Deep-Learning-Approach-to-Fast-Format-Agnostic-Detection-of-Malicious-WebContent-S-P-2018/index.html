A Deep Learning Approach to Fast, Format-Agnostic Detection of Malicious WebContent(S&P 2018)

发表于 2019-06-27 | 分类于 CCF-A |
字数统计: 1.7k 字 | 阅读时长 ≈ 6 分钟

作者介绍

一作：Joshua Saxe（Sophos的首席数据科学家）
公司：Sophos（一家英国软件和硬件安全公司）
个人主页：https://www.sophos.com/en-us/labs/researchers/joshua-saxe.aspx
Google Scholar：https://scholar.google.com/citations?user=OO7TDFsAAAAJ&hl=en
简介：Joshua Saxe是Sophos的首席数据科学家。他领导的数据科学团队专注于发明、评估和部署深度学习检测模型，以支持下一代端点安全解决方案。



1 概述

  本文提出了一种用于快速、不依赖于格式的检测恶意网页内容的深度学习方法：

  • 使用正则表达式直接从静态HTML文件（包括HTML, CSS, 嵌入式Javascript）中提取出与语言无关的令牌流，使其能够快速地在诸如防火墙和web代理之类的高频数据上下文中运行，适合部署到端点、防火墙和web代理；
  • 与诸如词袋模型等忽略空间信息的方法不同，该方法在层次空间尺度上检查内容，捕获局部性，从而产生更高的精度；
    实验达到 97.5%的正确率 和0.1%误报率（原因：之前未见过的网页内容），在普通硬件上以每秒100个以上的速度对小批量web页面进行分类。

2 系统结构

  • A feature extractor
     - 从HTML文档中解析出一系列令牌tokens。
  • Neural network model（Inspector与Master通过反向传播共同优化）
     √ Inspector
     - 在层次空间尺度上应用权重，并将关于文档的信息聚合到一个1024长度的向量中。
     √ Master network
     - 利用Inspector的输出，做出最终的分类决策。


2.1 特征提取

（1）使用正则表达式([ˆ\x00-\x7F]+|\w+) 对HTML文档进行划分，沿着非字母数字单词边界分割文档，得到tokens；
（2）将tokens分为16个长度相等的token块(其中长度定义为令牌的数量)，即每个块中的token数相同；
（3）将每个token块散列到1024个桶中，得到16x1024的向量表示。

2.2 Inspector

（1）将每个token块分解为不同的层次表示，得到31个聚合块输入前馈神经网中；
（2）前馈神经网由两个全连接层组成，每个层包括1024个神经单元，采用ReLU激活、Layer normalization、Dropout正则化；
（3）最后经过max pooling得到1×1024维向量。

2.3 Master

（1）Inspector输出的1×1024维向量作为输入；
（2）由两个全连接层组成，每层之前都有Layer norm和Dropout；
（3）最后一层由26个sigmoid单元组成，对应于文档的26个检测决策。其中一个sigmoid用于确定目标文档是恶意的还是良性的，其他25个sigmoid检测各种各样的信息标签，比如该文档是钓鱼文档还是攻击工具包的实例。

3 数据集

3.1 实验数据

  • 来源：威胁情报网站——VirusTotal
  • VirusTotal每天接收数以万计的新HTML文件，使用数十家安全供应商提供的60台web威胁扫描器对它们进行扫描。
  • Sophos订阅了VirusTotal的付费威胁情报服务，作为订阅的一部分，我们接收提交给VirusTotal的每个HTML文件及其相应的扫描结果。
  • 本文使用的实验数据来源于VirusTotal接收的2017年前10个月的数据，其中9.1日之前的数据用于训练，之后的用于测试。


3.2 标记策略

  • Benign HTML files：被0个安全供应商判定为恶意的；
  • Malicious HTML files：被>=3个安全供应商判定为恶意的；
  • Indeterminate：被1或2个安全供应商判定为恶意的，丢弃不用。

4 实验

  设计了两组对照实验，第一组测试了本文方法对其他模型的有效性，第二组探索了模型的内部工作原理。

4.1 模型比较

（1）LR-BoT: Elastic Net regularized logistic regression on bag of token features.
（2）FF-BoT: A feed-forward architecture using the 16284-length feature hashed bag of tokens feature representation used above.
（3）XGBoost-BoT: A gradient boosted decision tree (XG-Boost) model using the same feature input as FF-BoT.



结果：本文提出的模型的总体ROC曲线明显好于FF-BoT 和 XGBoost-BoT。

4.2 内部结构测试

（1）FlatSequential

  不使用求平均操作得到不同层次空间上的表示，而只使用最初的16个序列，测试通过在层次空间尺度上检测HTML文档所提供的性能增益。

（2）FlattenedFF

  使用相同的层次空间上的特征，但不采用在每步共享权重的检查器，而是简单将16×1024矩阵转化为1×16284的向量作为最后前馈网的输入。该实验测试使用共享权重检查器带来的性能增益。







结果：Proposed approach beats FlatSequential表明，在多尺度空间尺度上对内容进行检测，对提高检测精度至关重要；Proposed approach beats FlatSequential表明，inspector在检查每个空间上下文和尺度时使用相同的参数对于实现高检测精度至关重要，因为 FlattenedFF 为每个空间上下文使用单独的权重，但得出更差的结果。

4.3 结果细分

  为了更好地理解我们的模型学到了什么，采用恶意软件系列标签，并根据附加到哪些标签来细分真实的恶意软件样本。从表I中可以看出，最终模型在代码注入、浏览器攻击和iFrame操作攻击方面取得了最大的成功，而在钓鱼网站方面检测率最低。



5 总结

  使用深度学习能够学习到相对原始的web内容输入的高质量内部表示，其性能优于手工提取的特征。而且，从更广泛的意义上说，深度学习方法在检测恶意web内容方面具有广阔的前景。

6 个人观点

  • 优点：相对于传统使用机器学习检测恶意网页的方法，本文并没有通过手工的方式提取特征，二是将提取的token块经过哈希以后送入Inspector模块，得到1×1024维的向量表示，再输入到最后的Master网络进行分类。
  • 缺点：第一组对照实验对整个模型的效果进行了验证，第二组对照实验分别对多空间尺度特征和Inspector的效果进行了验证。但是本文并没有验证Master网络，没有跟其它网络进行对比。
