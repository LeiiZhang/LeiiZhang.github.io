2 概述

2.1 摘要

  本文是以前的工作的一个扩展，利用 favicon 和谷歌图像搜索来揭示网站的身份。身份检索技术包括一种有效的数学模型，可用于帮助从搜索结果的许多条目中检索正确的身份。本文介绍了基于 favicon 的钓鱼攻击检测的增强版本，引入了域名扩展特征（Domain Name Amplification feature）并添加了附加特征（addition features）。当被检查的网站没有 favicon 时，附加特征非常有用。

选择 favicon 是因为它代表了网站的品牌，此外，favicon 不会受到网页上显示的动态内容(如广告)的影响。

2.2 本文贡献

（1）利用从网站中提取的 favicon，通过谷歌图像引擎发现潜在的钓鱼尝试。与当前的反钓鱼解决方案不同，本文提出的方法不需要对基于文本或基于图像的内容进行深入分析，提高了检测速度；
（2）数学方程的使用使得能够从搜索结果的许多条目中检索真实身份；
（3）提出了 additional features ，以克服缺失 favicon 的问题；
（4）本文方法不需要图像数据库或任何来自合法网站的预存信息，因此降低了由于数据库过时而导致高错误检测的风险。

3 本文方法

3.1 Website Favicon

  favicon 是一个附加到 URL 的快捷图标，显示在桌面浏览器的地址栏或浏览器选项卡上或浏览器书签列表中的网站名称旁边。图 2 显示了 PayPal favicon 的 Internet Explorer 浏览器示例。图标代表一个16×16像素图像文件的网站标识，也能是 32×32，48×48，或 64×64 像素大小。



  获取 favicon：添加 favicon.ico 到网站域名后。例如，给定 PayPal 的 URL https://www.paypal.com/，提取域名 paypal.com，然后将 favicon.ico 附加到域名的末尾，使其成为 paypal.com/favicon.ico。将该新生成的 URL 输入地址栏中，就可以获取与 favicon 相关的信息。

3.2 Proposed Features

  总共有 5 个特征，从搜索结果中提取了 4 个（参见图 4 中所示的红色框）：
（1）Second-level domain (SLD)
  二级域名(SLD)：SLD是已知网站的名称，位于顶级域(TLD)旁边。例如，在 http://www.mydomain.com/ 中，mydomain 是 .com TLD 的 SLD。本文使用 SLD 作为特征是因为它有很大可能根据搜索结果显示网站的 identity。因此，从 GSI 返回的条目列表中提取 SLD。为了避免计数混淆，使用 “unique term” 表示从条目中提取的每个唯一 SLD，每个唯一项的频率由搜索结果中找到的 SLD 的出现次数计算。
（2）Path in URL (path)
  Path 通常位于 URL 的顶级域之后，例如，在 https://www.domain.com/image/index.php 中，/image/index.php 指向一个名为 index.php 的文件的唯一位置。使用 path 作为特征是因为搜索结果中通常包含与目标合法网站的标识相关的 path 中的 terms。虽然钓鱼者可以用不同的方式改变 URL 的路径，但必须在 URL 中保留 identity 关键字，以便让互联网用户相信他们访问的是正确的目的地。因此，从 GSI 返回的搜索结果中提取每个 URL 的完整路径，然后使用预先提取的 unique terms 从所有路径中找到匹配的标识。因此，为了捕获此属性，将记录路径中找到的每个 unique terms 出现的次数。
（3）Title and snippet (TNS)
  Title 是出现在 URL 顶部的文本，snippet 是出现在 URL 下方的描述。网站的标识并不总是出现在搜索条目的 URL 中，也可以在搜索条目的 title 或 snippet 中找到。因此，每个预先提取的唯一术语都用于在所有标题和搜索结果片段中找到匹配项。因此，记录在 title 和 snippet 中找到的每个 unique term 的出现次数。
（4）Highlighted text (HLT)
  高亮文本是出现在搜索结果的 title or snippet 中的粗体文本，突出显示的文本表明是搜索图标最相关和最重要的关键字。这一特征有很高的趋势显示的真正身份的图标，这是因为 favicon 的内容必须符合 GSI 定义的图像内容，以便在搜索结果中显示粗体文本。因此，该方法从搜索结果中的每个条目中提取粗体文本，根据预先提取的 unique term 找到匹配项，记录在粗体文本中找到的每个 unique term 的出现次数。

3.2 Additional Features

  一些网站没有 favicon，对此本文采用基于 url 的五个附加特征对网站进行分类。
（1）Suspicious URL

  • “@”符号：它强制删除左边的字符串，而右边的字符串被认为是实际的 URL。因此，用户将被重定向以访问位于“@”符号右侧的 URL。
  • “-”符号：钓鱼者通过在 URL 中插入“-”来模仿合法的域名，让用户相信这是合法的域名。

  基于“.”和“/”两个分隔符对 URL 进行 tokenize，通过遍历每个 token 找到“@”符号和“-”符号。如果令牌中有匹配的符号，则将该特征赋为1；否则，赋0。
（2）Dots in Domain
  在数据收集中发现，合法网页 URL 的域名中“.”的数量很少超过 5 个，而钓鱼网站则含有 5 个或 5 个以上的“.”。因此计算 URL 域名中“.”的数量，若含有 5 个或 5 个以上的“.”，则将该特征赋为1；否则，赋0。
（3）Age of Domain
  根据 APWG[23] 上发表的报告，钓鱼网站的最长寿命记录是 31 天。此外，报告还显示，大多数钓鱼网站的平均寿命只有一周。若寿命小于等于 30 天，则将该特征赋为1；否则，赋0。

[23] APWG, http://docs.apwg.org/reports/APWG Phishing Attack Report-Jul2004.pdf.

（4）IP Address
  使用 IP 地址创建网站比较廉价，因为它可以通过使用个人计算机作为 web 服务器来完成，钓鱼者不需要向任何域名注册商注册网站地址。因此，从查询网站提取 URL 并查找是否存在 IP 地址。若 URL 包含 IP 地址，则将该特征赋为1；否则，赋0。
（5）Web of Trust (WOT)



  WOT[24] 是一个根据互联网用户的反馈和来自第三方如 PhishTank 和 truste[25] 的信息显示另一个网站的声誉的网站。WOT 中有一个API，可以用来检查 Web 站点的合法性，图 6 显示了用于检索查询网站声誉的WOT API片段代码。代码中的 value 是插入查询网站的位置，api_key 是插入 api 注册 key 以激活 api 的位置。WOT 使用的声誉值如表 3 所示，其中 80 及以上的值表示该网站收到了非常好的网民反馈，而 19 及以下的值则表示该网站可能会危害到网民。为此，若美誉度小于 20，则将该特征赋为 1；否则，赋0。

4.1 数据集

4.2 实验

  通过 3 个实验验证了本文方法的有效性。（1）实验 1 在没有 additional 特征的情况下，对该方法的检测性能进行了评价；（2）实验 2 用于评估所提出方法与 additional 特征的集成，以便对缺少 favicon 的网站进行分类；（3）实验 3 比较了本文方法与其他钓鱼检测方法的性能。
  在实验 1 和实验 2 中，使用 7000 个网站（3500个合法网站和3500个钓鱼网站）进行训练和优化参数设置。在实验 3 中，总共使用了3000个网站（1500个合法网站和1500个钓鱼网站）来验证所提出的方法。值得一提的是，实验3中使用的3000个网站是实验 1 和实验 2 中没有使用过的一组新样本。实验结果采用以下测量指标：

  • True positive (TP)：钓鱼网站被正确分类为钓鱼网站；
  • True negative (TN)：合法网站被正确分类为合法网站；
  • False positive (FP)：合法网站被错误分类为钓鱼网站；
  • False negative (FN)：钓鱼网站被错误分类为合法网站；
  • F-score（$F_1$）：模型整体分类正确率，$F_1=\frac{2TP}{2TP+FP+FN}$。

（1）实验 1 —— 评估 Phishdentity
  实验 1 在没有 additional 特征的情况下，评估 Phishdentity 根据谷歌返回的搜索结果对网站进行分类的性能，使用 GSI API 中指定的默认参数来返回搜索结果。本实验设计了两个测试床：（1）第一个测试床是利用 SLD、path、TNS 和 HLT 对网站进行分类；（2）在第一个测试床所使用的所有特征的基础上，集成了一个 DNA 特征，形成第二个测试床，放大了一个与查询网站 SLD 对应的 unique term。



  如表 5 所示，第二个测试床在将 DNA 整合到测试中后性能得到改善，即使用 DNA 提高了 Phishdentity 查找网站身份的有效性。换句话说，第二个测试床能够定位正确的身份，即使搜索结果可能包含许多假身份。在对合法网站进行分类时，FP 减少了6.23%，但仍然很高，原因如下：（1）3500 个合法的网站中有 91 个没有 favicon，因此，假阳性总数占 2.6%；（2）当使用错误版本的 favicon 时，API 可能返回不正确的信息，当查询网站的 favicon 版本比存储在谷歌图像数据库中的更新时，就会发生这种情况。虽然这个问题造成了一些误报，但预计新的 favicon 将很快在谷歌图像数据库中可用；（3）在进行测试时，有 13 个网站限制了对它们的图标的访问。另一方面，第二个测试床在检测钓鱼网站方面也有一定的进步，它减少了 2.43% 的假阴性。假阴性数量的下降是由于谷歌能够过滤搜索结果中的恶意条目。但是谷歌需要一些时间来确定新的恶意条目的合法性，因为大多数钓鱼事件通常发生在攻击的早期阶段。采用第二个测试床进行后续实验。

（2）实验 2 —— 在没有 favicon 的情况下评估 Phishdentity
  实验 2 用于评估所提出方法与 additional 特征的集成，以便对缺少 favicon 的网站进行分类。
  设计了三个系列的实验：（1）第一个系列将用于确定 $C_1$ 和 $C_2$ 的最佳权重，使用 50 的阈值作为基线。每次记录 $C_1$ 增加 1 ，$C_2$ 减少 1 正确分类的网站的数量，在正确分类网站数最多时停止该过程。一旦确定了 $C_1$ 和 $C_2$，就用它们进行第二系列实验；（2）第二系列旨在确定最佳阈值 $\tau$，首先将 $\tau$ 设置为零，观察每次 $\tau$ 增加 1 时正确分类的网站的数量，在正确分类网站数最多时停止该过程；（3）第三个系列用于论证在 Phishdentity 中是否包含 additional 特征的性能差异，为第三个系列设计了两个测试床。第一个测试床用于证明没有 additional 特征时 Phishdentity 的性能，第二个测试床使用从第二系列实验中获得的 $\tau$ ，将 Phishdentity 与 additional 特征结合。实验二产生了 Phishdentity 的最终版本，并将其用于下一个实验。

（3）实验 3 —— 评估最终的 Phishdentity
  实验 3 比较了本文方法与其他钓鱼检测方法的性能，选择 CANTINA 和 GoldPhish 进行基准测试。CANTINA 是一种基于内容的反钓鱼方法，它使用 TF-IDF 技术，从网站中提取了 5 个最重要的关键词提供给谷歌搜索引擎，然后，从搜索结果中搜索匹配的域名以确定网站的合法性。另一方面，GoldPhish 是一种基于图像的反钓鱼方法，它利用了光学字符识别（OCR）技术。首先，它使用一个预定义的分辨率来捕捉网站的截图。然后，OCR从捕获的屏幕中提取文本信息，并将其提供给谷歌搜索引擎。最后从搜索结果中搜索匹配的域名，以确定网站的合法性。



  如表 10 所示，CANTINA 将 5.87% 的合法网站错误的归类为网络钓鱼，TF-IDF 对于一些合法的网站并不适用。如果查询网站包含非常少的文本信息来描述网站，那么它将为谷歌搜索引擎生成不正确的词汇签名，Zhang等人也讨论了这个问题，他们计划研究其他方法。从这个实验得到的证据来看，CANTINA 并不适合钓鱼网站。GoldPhish 在对合法网站进行分类时表现很差，非法分类的合法网站高达 19.87%。本文认为，当使用 OCR 工具从屏幕截图中提取文本信息时，GoldPhish 面临几个限制。首先，GoldPhish 使用一个固定大小的窗口来裁剪截图，这可能会排除位于裁剪区域之外的一些重要内容；其次，Alexa 的大多数网站在主页上都有广告，有些广告太大了，占据了截屏的一半。这导致 OCR 提取不正确的消息；第三，一些合法的网站在主页上使用很多图片，这可能导致 OCR 捕获的图像不包含有关网站的重要信息；第四，OCR 不能很好地处理一些不常见的字体和字体大小，这使得 OCR 无法正确识别字符。在这三种方法中，GoldPhish 对钓鱼网站的检测效果最好，为 98.07%，这可以归因于谷歌搜索引擎对钓鱼网站信息的返回很少或为零。

5 局限性和讨论

  本文提出的技术对实验有三个限制：
（1）钓鱼者可以稍微改变一个 favicon 的内容，从受害者的角度来看它仍是熟悉的，但它不会检索合法的网站。钓鱼者也可以替换一个类似于其他合法网站的图标。当谷歌还没有爬行到母公司及其子公司的所有新更新时，就会发生这种情况。然而，这个限制不是至关重要的。GSI 引擎将从更改后的 favicon 中提取不同的内容，并返回与目标合法网站无关的信息。此外，改变了 favicon 的网络钓鱼域名由于其相对较年轻，不太可能出现在搜索结果中。因此，改变了 favicon 的钓鱼网站仍然可以触发 Phishdentity 检测。

（2）对新的或不受欢迎的合法网站的错误分类。新网站可能不会被谷歌索引，WOT 可能没有关于这些网站的信息，因此该方法可能会将这些网站错误地归类为网络钓鱼。然而，随着时间的推移，最终新的网站将被列入 WOT 数据库。至于那些不受欢迎的网站，它们很可能不会成为钓鱼者的目标。此外，本文还提出了一种基于网站 URL 的分类方法。这个额外的方法是轻量级的，可以用来弥补 Phishdentity 无法对没有 favicon 的网站进行分类。这种 additional 方法将减少这种错误分类的概率，如实验 3 中的低假阳性所证明的。

（3）这些 additional 特征在一些合法网站上表现不佳。首先，WOT 将 Alexa 中列出的色情网站（pornographic websites）列为危险网站。虽然这类网站有很多恶意广告，会感染电脑病毒，但它在最终的计算中仍然被归类为合法的。理由是，我们不能否认，每一个合法的网站是无害的(即色情网站），但我们在这里的目标是确定合法性，而不是网站的危险。其次，WOT 将对那些不使用安全连接使用户登录或进行数字支付的电子商务网站给予较低的评级。在网页上使用安全连接可以防止窃听，但实施起来可能会很昂贵，尤其是对于只针对本地企业的电子商务网站。然而，这不大可能对合法网站造成虚假警报，因为合法性并不完全由 WOT 决定。
