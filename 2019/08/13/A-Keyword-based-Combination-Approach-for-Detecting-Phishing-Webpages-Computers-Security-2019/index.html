A Keyword-based Combination Approach for Detecting Phishing Webpages（Computers & Security 2019）

发表于 2019-08-13 | 分类于 CCF-B |
字数统计: 5.9k 字 | 阅读时长 ≈ 22 分钟

1 作者介绍

一作：Yan Ding（新疆大学研究生）

2 概述

  文本提出了 Search & Heuristic Rule & Logistic Regression (SHLR) 相结合的方法，用以 a）检测钓鱼网站常用的混淆技术；b）提高过滤合法网页的效率。具体方法如下：
（1）基于搜索引擎识别合法网页
  将网页的 title 标签的内容作为搜索关键词输入到百度搜索引擎中，如果该网页的域名与前10名搜索结果中的任何一个域名匹配，则认为该网页是合法的;否则，将执行进一步的评估。

个人观点：匹配并不能直接判断为合法网页，也有可能是 PSHCS（Phishing Sites Hosted on Compromised Servers），Jail-Phish中的方法是匹配且baseURL（protocol+hostname+pathname）相同才判定为合法网页。

（2）基于启发式规则识别钓鱼网页
  如果网页不能被识别为合法的，则根据字符特征所定义的启发式规则，进一步检查网页是否为钓鱼网页。前两步可以快速过滤网页，满足实时检测的需要。
（3）基于 logistic 回归分类器识别网页
  利用 logistic 回归分类器对剩余页面进行评估，提高检测方法的适应性和准确性。

  实验结果表明，SHLR 可以过滤 61.9% 的合法网页，基于 URL 的词汇信息可识别 22.9% 的钓鱼网页。

3 介绍

3.1 钓鱼策略

  合理的钓鱼攻击分类有助于采用有效的检测手段，本文根据钓鱼攻击中使用的关键过程对网络钓鱼攻击进行分类：（1）General phishing attacks：不针对特定对象的攻击；（2）Spear phishing attacks：攻击特定类别的对象（业务的关键信息持有者）；（3）Whale phishing attacks：针对公司首席执行官和政治领导人的钓鱼攻击。
  钓鱼网页的传播方式可以分为三类：（1）E-mail；（2）Short Message Service (SMS)；（3）Others：例如 Voice phishing，一种利用电话系统或IP语音(VoIP)系统进行攻击的技术。
  一种新的钓鱼攻击技术 domain shadow，攻击者窃取网站管理员的域帐户并创建成千上万的子域。然后，攻击者使用子域指向恶意站点或将恶意代码上载到域名服务器。

3.2 检测方法使用的特征的分类

  检测方法通过提取网页的相关特征来确定网站的性质，这些方法常用的特征可以分为以下几类：
（1）URL based
  1）URL
  许多不相关的词存在于钓鱼网页的网址中，因此，钓鱼url在其字符中与合法url存在一定的差异。
  2）Domain
  域名是由两个或多个单词组成的唯一标识符，它们之间用点隔开。能够清楚表示 URL 标识的词汇表被认为是 URL 的标识名称，比如 http://www.jd.com 中的 jd。钓鱼网页通常使用 URL 混淆技术。
（2）Host based
  1）Whois
  Whois是用于查询域的IP和所有者信息的传输协议。如果攻击者使用相同的电子邮件地址或名称来注册URL，则可以确定该网页是否是钓鱼网页。
  2）DNS
  DNS是域名和IP地址之间映射的分布式数据库。例如，可以使用DNS日志检测恶意网页、使用被动DNS分析技术检测恶意域名。
（3）Content based
  1）HTML
  HTML包含页面的所有资源，通常使用文档的长度、单词的数量和标签属性作为检测特征。
  2）Code
  用 JavaScript 编写的恶意代码可以在没有用户许可的情况下执行，可以使用 n-gram、entropy 和 word size 来检测 JavaScript 的混淆。

3.3 本文贡献

（1）为了检测在钓鱼网页中插入许多不相关单词的逃逸技术，使用网页的 title 标签的内容作为关键词，借助搜索引擎快速过滤合法网页。
（2）提出了七种用于检测 URL 混淆技术启发式规则，能够快速识别钓鱼网站。
（3）提出的基于关键字的组合检测方法具有良好的检测性能，能够满足实时检测的需要。

4.1 基于搜索引擎的检测阶段

  将网页 HTML 中 title 标签的内容作为搜索关键词，搜索引擎只用于识别合法的网页。实验表明，该阶段可以过滤 61.9% 的合法网页，错误率仅为0.04%，具有较好的识别可信网站的能力。

个人观点：匹配并不能直接判断为合法网页，也有可能是 PSHCS（Phishing Sites Hosted on Compromised Servers），Jail-Phish中的方法是匹配且baseURL（protocol+hostname+pathname）相同才判定为合法网页。

4.2 基于启发式规则的检测阶段

  通过字符串进行模式匹配可以检测 URL 混淆技术，例如URL命名标准违规和隐藏的网络钓鱼目标字。如果检测阶段可以直接判断URL是否是钓鱼网站，则可以省略分类器检测所需的特征提取过程，以满足实时检测的要求。下面详细介绍了三种分割URL的方式，不同的分割方法会产生不同的URL词汇表：

  • Mode 1：通过不包含下划线“_”和连字符“-”的非字母数字和非数字字符进行分割。
  • Mode 2：通过’ / ‘和’ \ ‘分割。
  • Mode 3：通过包含下划线“_”和连字符“-”的非字母数字和非数字字符进行分割。

  在合法 url 中，标识名（identification name）作为二级或三级域名标签使用，例如，在 http://www.taobao.com 中，taobao 是淘宝的标识名。phishing target word 是指钓鱼 URL 试图伪装的标识名称，钓鱼 URL 通过将 phishing target word 添加到其域名、路径或文件部分，以模仿合法网站。本文根据全球知名企业或集团(苹果、阿里巴巴、谷歌、Gmail)的标识名称，选择了电子商务、邮件通信、网上银行、APP 等领域的企业名称，如表 1 所示，建立了一个包含90个钓鱼目标词的词集。通过检测 URL 中的钓鱼目标词，可以确定它是否是钓鱼 URL 以及钓鱼网页试图伪装的主体，从而得到更有效的对策。



  由于标识名称的特殊性，合法网页不会在其 URL 中插入其他公司的标识名称。一种 URL 混淆技术是在 path 部分添加一个钓鱼目标词，这会导致多个标识名称出现在一个URL中。表 2 显示了 PhishTank 和Yahoo 数据集的URL总数以及各种异常信息的比例。6.9%的钓鱼url具有多个标识名，如http://www.frrtrt2.com/ Dropbox/id，这个链接是伪装成Dropbox的钓鱼URL。URL的路径部分包含钓鱼目标词dropbox，但不匹配域标签frrtrt2。基于这一现象，本文提出了规则1和规则2。

  • Rule 1：URL由模式1分割。如果路径中包含钓鱼目标词，且该词与二级或三级域名不同，则该URL被认为是钓鱼网站。
  • Rule 2：URL由模式1分割。如果域名或路径部分包含两个或多个不同的标识名，则该URL被认为是钓鱼网站。


  使用数字、分隔符和连接字符串伪造标识名的方法是攻击者使用的一种常见混淆技术。如表 2 所示，这种混淆技术的使用频率为7.1%。规则3可以有效地检测出使用数字进行的混淆。规则4是一个通过URL字重组的方法来检测更复杂的混淆技术。例如，图3为 URL 单词列表重组的流程图。在域名部分，标准顶级域名术语不参与单词的重组。此外，对于少于4个字符的名称，规则4不适用。

  • Rule 3：URL被模式1分割，但没有钓鱼目标词。若在删除 URL 中的数字后出现钓鱼目标词，则认为该网站是钓鱼网站。
  • Rule 4：URL被模式1分割，但没有钓鱼目标词。然后将 URL 按模式2划分为多个部分，并按模式3划分这些部分。在每个部分中，将连续索引位置上的单词连接起来形成新单词，并生成一个新的URL单词列表。如果在单词列表中存在钓鱼目标单词或存在包含其他标识名称子字符串的单词，则认为该网站为钓鱼网站。


  直接使用 IP 地址作为域名通常意味着链接指向一个私有页面，例如http://124.248.212.46/caixa/，这不是合法网页的URL，据此提出规则5。

  • Rule 5：若使用 IP 地址为域名，则该网站被视为钓鱼网站。

  在URL标准命名约定中，第一部分是协议类型，如 ftp 或 https。为了迷惑用户，钓鱼 URL 的非协议部分将编写协议字段，例如 http://www.oilchangeasheville.com/https:/www2.antander.com.br，试图冒充 Santander 的网站，据此提出规则6。顶级域名应该只出现在 URL 的域名部分，而不出现在其他部分，据此提出规则7。

  • Rule 6：协议字段出现在 URL 的非协议部分，则该网站被视为钓鱼网站。
  • Rule 7：顶级域名出现在 URL 的非域名部分，则该网站被视为钓鱼网站。

  通过规则库评估 URL 的性质，可以避免确定的钓鱼网站的特征提取过程。此外，使用逻辑回归分类器执行第三步检测避免了由于规则库的大小引起的钓鱼网站的漏报。

4.3 基于逻辑回归分类器的检测阶段

  使用 logistic 回归分类器，从DNS、Whois、与钓鱼词汇的相似度、词汇特征和HTML中提取特征，确定剩余页面的性质，提高检测方法的适应性和准确性。
（1）DNS doubt
  本文提出了一种新的检测特征——DNS doubt，它与 DNS 可信度成反比。首先计算 PhishTank(2776个钓鱼url)和 Yahoo(5883个合法url)数据集中的 url 使用的 DNS，DNS doubt 计算如下:
$$
F_{d}=\frac{P_{d n s}}{Q_{d n s}}
$$
其中 $P_{dns}$ 是在 PhishTank URL 数据集中使用 DNS 的频率，$Q_{dns}$ 是在 Yahoo URL 数据集中使用 DNS 的频率。如表3所示，$P_{domaincontrol.com}$ 为0.13，$Q_{domaincontrol.com}$为0.04。 因此，domaincontrol.com 的 DNS doubt 是 3.25。 对于 $Q_{dns}$，如果值为0，则无法正确解析URL；随后，DNS doubt计算如下：
$$
F_{d}=\frac{P_{d n s_{-} m a x}}{Q_{d n s_{-} m i n}}
$$
例如，$P_{dns_{max}}$ 为 0.13 (369/2823)，$Q_{dns_{min}}$ 为 0.0001 (1/8819)。

5.4 实验三

  实验 3 通过选择合适的机器学习分类算法，验证了基于LR分类器的检测阶段的有效性。采用数据组2，实验的主要目的如下：
（1）利用本文构建的特征集，比较 LR 与 L1 范数正则化、LR 与 L2 范数正则化、SVM、NB 的检测能力，并根据实验结果选择检测阶段使用的分类器。
（2）测试 DNS doubt 对 LR 分类器检测阶段的 accuracy、precision 和 recall 的影响。
  如表 9 所示，由于规则库的大小有限，基于启发式规则的检测阶段具有较高的假阴性，因此 SHLR 引入了分类器来提高检测方法的准确性和适应性。
  如表 8 和表 9 所示，前两个阶段的未识别 url 数量为4369，其中2138个是 PhishTank 的 url, 2231个是 yahoo 的 url。对于这部分URL，使用机器学习分类器进行第三次检测，执行10个 10-fold 的交叉验证来评估不同分类器的效果。如表 12 所示，在四个分类器中，L1规范正则化的LR更适合钓鱼URL检测。

5.5 实验四

  实验 4 综合分析了 SHLR 检测方法。选取了三种代表性的检测方法进行比较：PhishDef 、BeyondB 和BigData。BigData 只从URL中提取词法特性，BeyondB 和 PhishDef 从 URL 中提取词法和基于主机的特性，尽管 PhishDef 提取的特性更加详尽。这三种检测方法都使用词袋模型，实验的主要目的如下：
（1）使用数据组2分别比较分析三种方法检测的有效性，以及三种方法在不同组合下的有效性。
（2）使用数据组2比较 SHLR 与其他检测方法在 accuracy、recall、F-Score、precision、单个URL的平均特征提取时间、单个URL的平均识别时间、PRC曲线等方面的差异。然后，使用数据集3来测试 SHLR 的泛化能力。





  为了更好地分析 SHLR 检测方法，对三种检测方法分别单独使用和使用不同组合的方法进行了比较和分析。图 5 显示了其他检测方法的流程，如表 15 所示，SHLR在各项评价指标上均表现良好。
