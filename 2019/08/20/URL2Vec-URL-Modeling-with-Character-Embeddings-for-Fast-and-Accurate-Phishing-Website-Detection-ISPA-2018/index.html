URL2Vec: URL Modeling with Character Embeddings for Fast and Accurate Phishing Website Detection（ISPA 2018）

发表于 2019-08-20 | 分类于 CCF-C |
字数统计: 3.8k 字 | 阅读时长 ≈ 15 分钟

1 作者介绍

一作：Huaping Yuan（广东工业大学自动化学院）
Google Scholar：https://dblp.org/pers/hd/y/Yuan:Huaping



2 概述

2.1 摘要

  提出了一种基于深度学习的钓鱼检测方法。具体来说，在基于 word2vec 的词向量学习中，网站的 url 和这些 url 中的字符分别映射到文档和单词，将 character embedding 与 url 结构相结合，以获得 url 的向量表示。URL 被划分为以下五个部分：URL protocol, sub-domain name, domain name, domain suffix, 和 URL path。收集一个100万钓鱼检测数据集(1M-PD)，该方法对 1M-PD 数据集的 accuracy 为 99.69%，该方法在一台个人电脑上检测每个URL的平均速度为 32ms，比现有方法快得多，甚至可以被认为是实时的。

2.2 优点

（1）不需要提取任何手工制作的特征；
（2）只使用 url 而不访问 web 页面的内容，因此该方法易于使用，执行速度非常快（在毫秒内），可用于实际处理大规模数据集。

2.3 本文贡献

（1）本文提出了一种新的框架，利用 characters 在 url 中的 embedding 表示来检测钓鱼网页。word2vec 模型实现的 character embedding 不依赖于任何外部知识、手工信息或网络负载，颠覆了传统方法；
（2）基于目前已有的 character embedding 方法，设计了一种基于结构的 URL 统一表示策略。特别是将 URL 划分为五个部分，包括 URL protocol, sub-domain name, domain name, domain suffix, 和 URL path，这对性能有很大的贡献；
（3）发布了一个由 100 万个 URL 组成的钓鱼数据集 1M-PD，有助于促进钓鱼检测应用的研究。

3 URL 建模与初探

3.1 问题定义

  假设有一个语料库 $C$，包含 $N$ 个 url，即 $C= {u_1, u_2, \dots, u_N}$。对于一个URL $u_i$，它由五个部分组成，即 $u_i=(S_1S_2S_3S_4S_5)$，每个部分都可以看作是字符序列，如大写字母、小写字母、数字等。例如，有 $S_1=(c_1, c_2, \dots, c_i, \dots, c_n)$，其中 $c_i$ 表示一个字符，n 是一个变量，表示该部分的长度。URL 的结构示例如图 1 所示。



  本文提出通过语言模型来学习 url 中字符的向量表示，进而得到 url 中每部分的向量表示，最后得到 url 的向量表示。

3.2 语言模型初探

  本文提出了一种通用的语言建模方法来探索 url 中的 character embedding，其灵感来自于DeepWalk[21]。更具体地说，对于包含字符序列的URL，
$$
S_u=(c_1, c_2, \dots, c_n)
$$
direct analog 是在给定 URL 中所有先前字符的情况下估计观察字符 $C_n$ 的可能性，希望通过训练 URL 语料库。最大化以下概率：
$$
Pr(c_n|c_1, c_2, \dots, c_{n-1})
$$
  自然语言处理的任务是找到单词的分布式表示形式。因此，引入一个映射函数 $\phi$ 来表示 URL 中每个字符 $c_i$ 的潜在向量表示。问题变成了估计：
$$
Pr(c_n|\phi(c_1), \phi(c_2), \dots, \phi(c_{n-1}))
$$
  或者，用一个词来预测上下文[22]，导致以下优化问题：
$$
min -logPr({c_{i-w}, \dots, c_{i-1}, c_{i+1}, \dots. c_{i+w}}|\phi(c_i)) （4）
$$
  通过式（4）中的优化问题，可以构建URL上字符间共享相似性的表示，并泛化机器学习任务。

[21] Perozzi, Bryan, Rami Al-Rfou, and Steven Skiena. “Deepwalk: Online learning of social representations.” Proceedings of the 20th ACM SIGKDD international conference on Knowledge discovery and data mining. ACM, 2014.
[22] T. Mikolov, I. Sutskever, K. Chen, G. S. Corrado, and J. Dean. Distributed representations of words and phrases and their compositionality. In Advances in Neural Information Processing Systems 26, pages 3111–3119. 2013

4 基于 URL character embedding 的钓鱼检测

  本文的钓鱼检测系统由三个模块组成，如图 2 所示。Character Embedding Learning 模块实现了 url 中 characters 的向量表示。给定 character embeddings，URL Vector Representation 模块通过将 url 的五个部分中的字符的平均 embeddings 连接起来获得 URL 的向量表示。最后，检测器模块根据 url 的向量表示训练机器学习算法，将 url 分类为钓鱼类和合法类。



4.1 Character Embedding Learning

  所需的输入是语料库和词汇表，输出是词汇表中的单词的 embeddings。词汇表可以看作 URL 中的字符，例如，大写字母，小写字母，数字等。为了获得 URL中 characters 的 embeddings，本文采用 SkipGram 根据目标词来预测上下文。在随后的实验中，将研究利用不同策略进行 character embedding learning 的变化，并评估其对学习效果的影响。事实上，选择不同策略的影响很小，特别是对于大规模数据。以 SkipGram 为例进行说明：

（1）SkipGram
  SkipGram 是一种语言模型，其目的是最大化单词在句子中出现在窗口 $w$ 中的共现概率。对于 URL 中的每一个 $c_i$ 和 $c_k \in u[i-w: i+w]$，将 $c_i$ 映射到其当前的表示向量 $\phi (c_i)$。给定 $c_i$ 的表示，将其邻居的后验概率 $Pr(c_k | \phi (c_i))$ 最大化，这种后验分布可以通过使用相当多的分类器来学习。考虑到计算的复杂性，使用 Hierarchical Softmax 来近似概率分布。

（2）Hierarchical Softmax
  给定 $c_k \in u$，计算 $Pr(c_k | \phi (c_i))$ 是不可行的。如果将字符分配到二叉树的叶子上，预测问题就变成了最大化树中特定路径的概率。

其中 $Pr(c_k | \phi (c_i))$ 可由分配给节点 $b_l$ 父节点的二进制 calssifer 计算。因此，它将计算复杂度从 $O(|V|)$ 降低到 $O(log|V|)$。并加快训练过程。

（3）Optimization

4.2 URL Vector Representation

  给出每个 character 的 embedding 向量，通过对 URL 中每个 part 中的 characters 的 embedding 向量进行平均，得到每个 part 的特征向量。然后将 5 个 part 的 embedding 向量连接起来，得到统一的 URL 向量表示。对于某些 url 的 part 小于 5 的，将与缺失部分对应的向量中的所有元素设为零。例如，假设得到了 10 维 character embeddings，通过平均运算，每个 part 的向量表示都为 10 维。最后，通过将五个 part 的表示连接在一起，URL 的向量表示为 50 维。

4.3 Detector

  给定 url 的向量表示，任何分类算法都可以无缝地应用于检测钓鱼 url，本文采用 XGBoost。在实验中，对梯度增强决策树(GBDT)、KNN、Logistic回归(LR)、随机森林(RF)、决策树(DT)等分类模型的性能进行了比较研究。为了完整起见，总结了本文提出的基于 character embedding 的钓鱼检测方法，如 Algorithm 1 所示。

5.2 Baselines

  Baselines 包括四类：1）第一类包括一些公开数据集或开源代码的方法，如WTF[27]、PTD[30]等，可以进行比较；2）第二类包括一些代表性的方法，如VS [28]， CW[29]，但它们没有公共数据集或开源代码，因此，此类方法只能用于水平比较；3）第三类是本文方法的变体，在字符嵌入组件中使用不同的语言模型，记作LMV，目的是找到更适合当前任务的语言模型；4）第四类包括本文方法的变体，在检测器组件中使用不同的分类器，记作CMV。四类基线如下所示：

（1）Word2vec and term frequency features (WTF，Word2vec 和 词频特性)
  Farhan 等人[27]将 URL 的 word2vec 特性和 TF-IDF 特性结合起来用于钓鱼 URL 检测，特别采用 logistic 回归(LR)、支持向量机(SVM)和卷积神经网络(CNN)进行分类。

（2）Phishing detection through phishing target discovery (PTD，通过钓鱼目标发现钓鱼检测)
  Liu 等人[28]提出发现给定网页的寄生群落，并测量网页与给定网页寄生关系的强弱。此外，利用寄生关系强度阈值来识别钓鱼网页。

（3）Visual similarity-based approach (VS)
  Liu等人[29]提出通过对相关网页进行聚类来识别给定(可疑)网页的钓鱼目标。利用链接、排名、文本相似度、网页布局相似度等关系进行聚类。

（4）Confidence weighted approach (CW)
  Blum 等人[30]利用置信加权分类结合内容进行钓鱼 URL 检测。作者开发了一个动态和可扩展的系统，用于检测现有和新兴类型的网络钓鱼域。

（5）Language model variants (LMV)
  Character embedding component 可以采用不同的语言模型，如 Skip-gram，CBOW，Doc2vec，GloVe 来学习 URL 中字符的向量表示。

（6）Classifier model variants (CMV)
  Detector 可以采用不同的分类算法，如 logistic 回归(LR)、梯度增强决策树(GBDT)、随机森林(RF)等，来识别合法或钓鱼 url。

[27] Abdi, Farhan Douksieh, and Lian Wenjuan. “MALICIOUS URL DETECTION USING CONVOLUTIONAL NEURAL NETWORK.”
[28] L. Wenyin, G. Liu, B. Qiu, and X. Quan, “Antiphishing through Phishing Target Discovery,” in IEEE Internet Computing, Vol. 16, no. 2, 2013, pp. 52-61.
[29] G. Liu, B. Qiu, L. Wenyin, “Automatic detection of phishing target from phishing webpage,” in Pattern Recognition (ICPR), IEEE, 2010, pp. 4153-4156.
[30] A. Blum, B. Wardman, T. Solorio, and G. Warner, “Lexical feature based phishing URL detection using online learning,” in Proceedings of the 3rd ACM Workshop on Artificial Intelligence and Security, 2010, pp. 54-60.

5.3 Comparisons

（1）在公共数据集或开源代码上的比较 WTF 和 PTD



  WTF 在他们的论文中报告了其在 420K-PD 数据集上的表现，将本文方法与 WTF 进行了比较，如表 3 所示。 从表中可以看出，本文方法在精确度，召回率和 F1 值方面优于 WTF 的变体。实验结果表明，本文对 URL 的建模比 WTF 提取的 URL 特征更有效。



  PTD 的源代码是公开的，直接使用他们的代码。然而，PDT 需要访问 URL 的内容，本文从 Phishtank 中随机选择了 574 个钓鱼 url 和 Alex 发布的 2752 个合法 url 进行评估。结果如表 4 所示，本文方法取得了较好的性能。$N_{L \to L}$ 为正确分类的合法 url 个数，$N_{P \to P}$ 为正确分类的钓鱼 url 个数，$N_{L \to P}$ 为错误分类的合法 url 个数，$N_{P \to L}$ 为错误分类的钓鱼 url 个数。

（2）水平比较 VS 和 CW
  将本文方法与 VS、CW 在他们论文中报道的结果进行比较，直接采用他们的结果进行比较有以下几点原因：1）他们实验中使用的基线数据集还没有公开；2）他们的基线方法必须访问 web 页面的内容，而钓鱼方法通常生存的时间很短，甚至只有几个小时，通过从 web 内容中提取手工制作的特征很难完全实现它们；3）基线方法可能依赖一些第三方服务，而这些服务可能已经不提供了。



  实验结果如表 5 所示。CW 需要从 url 和网页内容中手工提取特征，依赖于对大量网页的分析，导致了巨大的时间成本。此外，它们的性能是在相当小的数据集中报告的。而本文提出的基于字符嵌入的方法在大规模数据集上的多指标性能最高，不仅不依赖于任何手工特性，而且具有更低的时间成本。实验结果表明了该方法的有效性和高效性。

（6）评估 Embedding 的维度








  如图 7 和图 8 所示，性能随着维数的增加而提高，维数在 250 之后会下降（即 character embedding 的维数为 50）。实验结果表明，嵌入相对低维或高维的字符并不是最好的。一个相对较低的维度可能不足以表达字符的语义，而一个较高的维度可能包含一些用于钓鱼检测的无效信息，增加了计算复杂度。

6 总结

  本文提出了一种 character embedding 模型来自动实现 url 的向量表示，不需要设计任何手工制作的特征，且不需要网络访问。该方法能在几毫秒内检测到钓鱼网站，并在大规模数据集上取得较高的性能。
