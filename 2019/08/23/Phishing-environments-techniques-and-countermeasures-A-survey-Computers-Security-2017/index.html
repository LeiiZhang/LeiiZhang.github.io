Phishing environments, techniques, and countermeasures: A survey（Computers & Security 2017）

2 概述

2.1 摘要

  本文调查了钓鱼攻击和反钓鱼技术的发展，不仅在传统的环境（如电子邮件和网站），而且包括新的环境（如移动和社交网站）。从整体观点出发，提出了一种分类方法，包括攻击技术、对策、目标环境和通信媒体。该分类法不仅将为在各种环境中设计有效的钓鱼检测和预防技术提供指导，而且还将帮助从业人员评估和选择工具、方法和特征来处理特定类型的钓鱼问题。

2.2 现有的钓鱼分类法



  表 1 总结了现有钓鱼分类法的覆盖范围，本文的调查旨在通过提出一个新的分类法来解决以前分类法的局限性。

3 新的钓鱼分类法



  如图 2 所示，在该分类中，网络钓鱼攻击被描述为四个维度：传播介质、目标环境、攻击技术和对策。

  • 传播介质：E-mails, Websites, IM, Online Social Networks, Blogs and Forums, Mobile, Voice over IP；
  • 目标环境：PC, smart devices, voice devices, Wi-Fi Decices；
  • 攻击技术：基于目的将攻击技术分为攻击初始化、数据收集和系统渗透。
  • 对策：分为机器学习、文本挖掘、Human Users、配置文件匹配、Others 五大类。Others 类别进一步细分为 Ontology、蜜罐、搜索引擎和基于客户机服务器的身份验证。

3.1 Machine learning

  相关技术主要分为三类：分类、聚类和异常检测，如表 H 所示。



（1）分类
  根据用于发现网络钓鱼的特征类型（见附录H），钓鱼分类器可以分为三类：基于 URL特征、基于文本内容特征、基于混合特征。
（2）聚类
  聚类方法将实例划分为钓鱼和合法集群，聚类的目的是根据对象的相似性对其进行分组。如果将每个对象表示为一个节点，并根据不同对象的共同特征度量它们之间的相似性，则可以使用聚类算法来识别具有相似观测值的(节点)组。可以选择组的数目，使同一组中的节点具有比不同组中的节点更高的相似性，集群结构信息反过来又用于将新对象分配给正确的集群。聚类方法的两个重要组成部分是两个数据样本之间的相似性（距离）度量和聚类算法。
（3）异常检测



  异常是数据中与正常行为模式不一致的模式，基于异常的钓鱼检测方法本质上将钓鱼视为异常值。每个网站都宣称自己在网络空间中拥有独特的身份，或明或暗。当网络钓鱼网站恶意声明虚假身份时，与合法网站相比，它总是表现出不正常的行为，如网页和 HTTP 事务中的一些 DOM 对象。异常检测算法通过捕获这些异常来发现钓鱼网站，将钓鱼信息的特征与一个或多个最近邻居的特征进行比较，对被分析的可疑信息进行打分。如果异常得分超过一个分界点，该网页将被归类为钓鱼。钓鱼网页中的异常线索如表 2 所示。



  表 3 总结了基于机器学习的钓鱼检测方法、特征、传播介质和对应的文章。结果表明，分类是网络钓鱼检测的主要方法。此外，值得注意的是，电子邮件中的钓鱼检测大多依赖于基于内容的特征，而网站中的钓鱼检测则依赖于基于 url 的特征。此外，研究表明，将聚类技术应用于网络钓鱼的研究相对较少。在网络钓鱼聚类中，对图像、语音等特征的研究还比较少。此外，异常检测技术已被用于检测 IM 以及网站和电子邮件中的网络钓鱼。

3.2 Text mining

  文本挖掘是指利用数据挖掘和机器学习技术从文本中发现趋势、模式或有用的知识，通过分析可疑材料的模式，包括但不限于电子邮件、网站、url、即时消息和 SMS 的内容，来识别网络钓鱼。文本挖掘技术在网络钓鱼检测中有三种类型：词频逆文档频率(TF-IDF)、正则表达式(RE)、潜在语义分析(LSA)和主题模型。



  表 4 总结了基于文本挖掘的钓鱼检测研究。从表中可以看出，电子邮件钓鱼检测利用了 LSA 技术。相比之下，网络钓鱼的检测主要集中在 TF-IDF 技术上。这可能是因为网页 url 中缺少上下文以及页面内容的广泛多样性。

3.3 Human users

  Human-based 对策和行为因素表征谁陷入网络钓鱼，对于防止网络钓鱼攻击非常重要。用户研究旨在衡量用户对网络钓鱼材料的反应，考虑了行为因素和人口统计因素。这些研究通常涉及用户进行网络钓鱼识别，大多数用户研究都是使用反网络钓鱼系统进行的。然而，其中一些提供了在面对网络钓鱼攻击时提高用户意识或让他们参与网络钓鱼检测机制。



  表 5 总结了基于用户的钓鱼检测方法。从表中可以看出：（1）用户教育是预防手机钓鱼最常用的对策之一；（2）用户教育、培训和投票方式在 IM、社交网络等多种传播媒体中还有待探索；（3）用户投票可以在用户感知类别下交叉列出。

3.4 Profile matching

  配置文件匹配利用有关域名的信息、用户最近访问的域名、在这些域中的凭证、被访问域的其他特征（例如布局和图像）来创建基于特征的配置文件，并使用它们来检测钓鱼。配置文件匹配组件可以很简单（例如URL匹配），也可以包含复杂的技术（例如图像匹配），浏览器警告和大多数钓鱼工具栏都属于这一类。有几种工具可用来以分散的方式管理用户配置文件(Florencio和Herley, 2006;Lee等人，2008年)，并巩固他们的身份(如OpenID，自由联盟的SAML，微软的WS)。将依赖于配置文件匹配策略的钓鱼策略分为四类：使用历史匹配、模式匹配、视觉结构匹配、白名单匹配和黑名单匹配。



  表 6 总结了配置文件匹配对策。使用历史记录匹配方法不仅适用于欺骗性网站，还适用于社交网站中的网络钓鱼电子邮件和欺骗性用户配置文件。然而，这些方法主要用于检测网站中的网络钓鱼。 除了网站欺骗之外，其他类型的配置文件匹配方法还没有被应用。

3.5 Other types of countermeasures

  几种新兴的反钓鱼技术，包括 Ontology、蜜罐、搜索引擎和 Client–server 身份验证。



  表 7 总结了 Other 类别中的四种对策。其中，只有基于搜索引擎的解决方案被用于检测社交网站中的网络钓鱼，而本体仅用于电子邮件。相比之下，蜜罐已经被用来收集各种媒体上关于钓鱼者的信息，如 IM（例如收集钓鱼者用来发送钓鱼材料的账户）、IP 语音（例如创建可疑语音来源的黑名单数据库）和网站。Client–server 认证策略主要用于防止电子邮件和网站环境中的网络钓鱼。

5 钓鱼检测的未来研究问题

  如表 8 所示。



（1）Zero-day 钓鱼检测
  解决 Zero-day 网络钓鱼方面的一些开创性的工作（Moghimi and Varjani, 2016; Zhan and Thomas, 2011）。然而，其中一个被忽视的方法是与已知网络钓鱼模式的上下文相似性。尽管网络钓鱼具有其独特的特点，但它与已知的网络钓鱼有着共同的语境关系。因此，上下文相似性可以通过预测对手未来可能的活动和可能采取的路径来预测未知的钓鱼模式。检测未知钓鱼企图的另一种有前途的方法是将异常检测与上下文相似性相结合。在这种组合方法中可以使用多种异常检测方法，例如基于异常的一类分类方法，这些方法最近被用于检测 zero-day 入侵。

Moghimi M, Varjani AY. New rule-based phishing detection method. Expert Syst Appl 2016;53:231–42.
Zhan J, Thomas L. Phishing detection using stochastic learning-based weak estimators. In: IEEE symposium on computational intelligence in cyber security (CICS’11). IEEE; 2011. p. 55–9.

（2）Multi-stage 钓鱼检测
  多阶段攻击在一种通信媒体中初始化，在另一种通信媒体中完成。因此，有必要实现新的钓鱼解决方案，在所有阶段进行跟踪和检测。例如，当电子邮件将用户重定向到受欺骗的 web 页面时，需要使用适当的工具同时检查两种媒体中的钓鱼模式。联合聚类（co-clustering）是同时对两类对象进行分类的聚类方法之一，通过同时聚类钓鱼 e-mail 中的类似模式以及 e-mail URL 引用的相应网站内容，可以使用联合聚类来创建二分图（bi-partition graphs）。

（3）欺骗性语音钓鱼检测
  语音钓鱼检测在网络钓鱼研究中受到的关注有限，到目前为止，小波聚类等几种机器学习技术具有很大的潜力，小波聚类依赖于聚类语音波来进行语音识别。由于小波聚类是一种无监督的技术，因此可以用于检测未知的语音钓鱼模式。此外，蜜罐可以用来收集可疑的电话号码，创建黑名单。未来的研究方法需要专注于通过分析蜜罐收集的可疑语音的模式来创建基于语音的分类技术。此外，在移动环境中检测语音网络钓鱼也是一种趋势。

（4）自适应环境下的钓鱼检测
  适应性环境是指那些根据其使用情况（如网站、移动应用程序）在战略或战术上进行修改的环境。现有的反钓鱼技术在这样的环境下使用可能会导致很高的假阳性率，因为它们大多是基于规则的。为了解决这个问题，可以使用自适应分类器。此外，随着网站结构的变化，由特定网站的概念组成的本体可以处理网站拓扑结构的构建和重构。具体来说，可以利用站点本体在被检查站点的结构与存储的本体之间进行映射。然后，根据被检查的网站与合法网站本体在结构和内容上的相似性，利用适当的分类功能对网站进行分类。

（5）基于排名的钓鱼检测
  基于排名的对策不仅可以用于检测单页钓鱼，还可以用于检测使用类似域名和特征的钓鱼 farms。尽管搜索引擎在检测网络钓鱼方面显示出了有效性，但验证环境是外部的，因此效率低下。具体来说，检测应用程序必须向搜索引擎提交可疑 url，评估搜索结果，并将结果传回钓鱼检测引擎。此外，用于检查发布到搜索引擎的 url 的 robots 通常被认为是搜索引擎的垃圾邮件尝试。为了解决这些问题，建立一个基于内部排名的机制是钓鱼检测的一个有前途的方向。

（6）多语种钓鱼检测
  根据涉及国家的不同，处理欺诈威胁在许多情况下意味着不同语言之间的差异。多语言 IR 是目前研究的一个新领域，主要致力于建立语言无关的 IR 模型。其他几种基于语义的技术，如 ontology、LSA 和语义网络在各种应用中都取得了成功，这些技术可以扩展到不同语言的钓鱼检测。

（7）检测配置文件克隆攻击
  社交网站固有的诸多问题之一就是个人资料的克隆。在此类攻击中，虚假的用户资料被创建为同一或不同社交网络上真实用户的副本。克隆攻击者的主要目的是误导用户的朋友与伪造的个人资料建立虚假关系。攻击者可以利用这种信任来收集用户朋友的个人信息，并进行各种类型的网络诈骗。Profile Trust 模型是一个很有前途的解决方案，它通过评估用户收到的材料来工作(Chou et al., 2004; Wang et al., 2010)。例如，可以根据好友数量、社交网络使用历史、关注者数量等来验证发送者的 Profile。

Chou N, Ledesma R, Teraguchi Y, Boneh D, Mitchell JC. Client-side defense against web-based identity theft. In: 11th annual network and distributed system security symposium (NDSS’04). San Diego, CA: 2004.
Wang W, Zeng G, Tang D. Using evidence based content trust model for spam detection. Expert Syst Appl 2010;37(8):5599– 606. doi:10.1016/j.eswa.2010.02.053.

（8）基于上下文的交互式网络钓鱼预防
  传统安全训练解决方案的设计大多没有考虑到用户的上下文因素。此外，大多数用户并不注意在他们的上下文中关于钓鱼的警告信号。因此，开发上下文感知的交互式网络钓鱼检测和预防方案具有重要的现实意义。用户语境不仅包括与用户直接相关的因素，还包括传播媒介和钓鱼目标的特征。
