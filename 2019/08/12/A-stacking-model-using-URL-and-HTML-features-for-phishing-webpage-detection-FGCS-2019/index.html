A stacking model using URL and HTML features for phishing webpage detection（FGCS 2019）

发表于 2019-08-12 | 分类于 CCF-C |
字数统计: 4.9k 字 | 阅读时长 ≈ 18 分钟

1 作者介绍

一作：Yukun Li（广东工业大学自动化学院研究生）



2 概述

  本文设计了轻量级的 URL 和 HTML 特征，引入了 HTML string embedding。此外，还将 GBDT、XGBoost 和 LightGBM 多层结合，设计了一个 stacking 模型，使得不同的模型可以互补，从而提高了钓鱼网页检测的性能。本文收集了两个用于评估的真实数据集，分别为 50K-PD 和 50K-IPD。50K-PD 包含 49,947 个带有 URL 和 HTML代码的网页；50K-IPD 包含 53,103 个网页，除 URL 和 HTML 代码外还有 Screenshots。本文贡献如下：
（1）在特征提取方面，将人工设计的特征与 Word2Vec 模型提取的 HTML string embedding 相结合；
（2）结合几种机器学习模型的优点，提出了一种检测钓鱼网页的 stacking 模型；
（3）设计了一个实时的钓鱼检测系统，可以用来保护用户免受钓鱼攻击；
（4）发布了一个真实的钓鱼网页检测数据集。

3 本文方法

  本文方法概述如图1所示。首先，从网页的 url 和 HTML 代码中提取特征，并将它们连接为特征向量。此外，还设计了 stacking 模型进行预测。



3.1 Feature extraction

3.1.1 URL-related features

  提取了8个 URL-related 特征，其中前6个特征由同行研究者设计。这些特性不依赖于第三方服务，可用于开发 real-time 系统。

  • IP address：钓鱼网页的域名通常是 IP 地址，如 http://62.141. 45.54/portaleTitolaris8/;
  • Suspicious symbols：分别计算“@”、“−”和“∼”三个符号的数量;
  • https：二进制值用于指示 URL 的协议是否是“https”;
  • URL length information：长度信息分别包括 URL 中的字符数及其域名;
  • Number of dots in domain name：网络钓鱼网页往往在 url 中使用更多的点，域名中的“.”的数量用于表示此特征;
  • Sensitive vocabulary：‘‘secure’’, ‘‘account’’, ‘‘webscr’’, ‘‘login’’, ‘‘ebayisapi’’, ‘‘signin’’, ‘‘banking’’, ‘‘confirm’’，使用URL中这些单词数量的统计数据作为特征;
  • Top-level domain name related：Phishing URLs 通常包含多个顶级域名，例如“http://www.ebay. com.urgd.com/path”。 因此，本文设计了与顶级域名对应的三个特征：顶级域名是否在 Stuffgate（http://stuffgate.com/）的顶级域名列表中，域名中的顶级域名数量，以及 URL 路径中的顶级域名数;
  • Similar target brands：根据’.’和’/‘拆分 URL 以提取字符串，计算每个字符串与来自 Phishtank 的给定目标品牌之间的 Levenshtein 距离。如果存在小于2的距离值，将此特征设置为1；否则，将此特征设置为0。

3.1.2 HTML-related features

  提取了12个 HTML-related 特征，本文设计了其中后6个特征；提出使用 Word2Vec 模型来学习 HTML string embedding。

  • Number of internal and external links：为了欺骗用户该网页是合法的，网络钓鱼网页通常使用外部资源，如其钓鱼对象来丰富内容，导致网络钓鱼网页内部链接很少，外部链接较多。从HTML代码中的所有链接中提取域名，并将其与网页URL的域名进行比较，以识别内部和外部链接。将内部链接的数量作为一个特征，外部链接的数量作为另一个特征;
  • Empty link：钓鱼网页通常使用空链接来伪装拥有大量超链接，有两种空链接，即‘‘ ’’ 和 ‘‘’’，网页中的空链接数用于表示此特征;
  • Login form：首先，必须识别网页中所有的 <form> 标签。此外，对于标签 <form> 中的每个子标签 <input>，将使用关键字 “password” 和 “pass”，或 “login” 和 “signin” 进行匹配，以确定是否存在登录表单。
  • Length of HTML content：钓鱼网页的目的是欺骗用户的登录信息，所以其 HTML 内容通常比较简单，即钓鱼网页的 HTML 代码长度通常比合法代码的长度短。将 HTML 内容的长度细化为 tag 内容的长度，选择了5个标签分别计算它们的长度，包括 <style>， <script>， <link>， <!−−> 和 <form>。标签 <style> 和 <link> 主要设置页面样式和 CSS，而钓鱼网页通常为了快速开发而没有这样的样式；标签<script>用来动态改变网页内容，而钓鱼网页通常是静态的；标签<!−−>用于注释，而钓鱼网页通常是一次性开发，开发人员不执行二次维护通常不写注释；标签<form>用于在网页内设置表单。每个标签的长度分别作为一个特征，此外，HTML 代码的长度也作为一个特性，总共提取了6个特性来表示 Html 内容的长度。
  • Alarm window：一些钓鱼网页会弹出一个报警窗口，供用户输入个人信息，使用二进制值来表示网页是否有报警窗口。
  • Redirection：一些钓鱼网页设计人员创建供用户访问的合法网页，这些网页将重定向到钓鱼网页。重定向的仿冒网页在 HTML 代码中使用”redirect”字符串。使用二进制值表示是否存在此类字符串。
  • Hidden/Restricted information：HTML 代码中的一些特殊代码可以防止内容显示或限制标签的功能，这可能会被钓鱼网页使用。这些特殊代码适用于特定的标签，总结如下。对于这三个方面，可以相应地生成三个特征。例如，HTML 代码中所有 tag “div” 中包含的特殊代码的数量作为一个特征。


  • Consistency between title brand and URL brand：一般来说，合法网页的标题包含其 brand 名称。如果网页的 brand 与 URL brand 不一致，很可能是网络钓鱼。
  • Consistency between the most frequent link brand and URL brand：直观上，由于目标网页资源的广泛使用，最常见的链接 brand 应该与钓鱼网页的 URL brand 不一致。如图2所示，提取 HTML 代码中的所有链接，并计算每个 brand 名称在链接中出现的次数，得到一个 brand 名称字典。此外，基于 URL brand 是否击中最频繁的链接 brand 来计算一致性。此外，使用链接中出现频率最高的链接 brand 的次数作为特征。


  • Internal and external resources：基于网络钓鱼网页往往使用外部资源的思想，分别使用“<link>”、“<img>”、“<script>”和“<noscript>”四个标签来计算作为特征的内部资源和外部资源的数量。
  • Number of the URL brand name appears in HTML code：由于频繁使用内部资源，合法网页的 URL brand 名称通常在 HTML 代码中出现多次，钓鱼网页的情况通常相反。
  • HTML string embedding：Word2Vec 是一种神经网络语言算法，专注于学习单词的分布式表示，在 Word2Vec 中，有输入层、投影层和输出层。在 Word2Vec 的训练阶段，给定目标词的上下文，训练模型以预测目标词作为输出。在训练 Word2Vec 模型之后，可以提取投影层的参数来表示单词。本文基于空格提取 HTML strings，然后学习 n 维的 HTML string embedding，最后对每个 string embedding 求平均，得到一个表示 HTML 文档的 n 维向量，流程如图3所示。


3.2 Stacking model



  本文提出的两层 stacking 模型如图4所示，每层由三个基本模型组成，即，Gradient Boosting Decision Tree(GBDT)，XGBoost，LightGBM。更具体地说，使用类似于 K-fold 交叉验证的方式来训练基本模型。将训练集分为 Z 份，其中 Z-1 份用于训练，1 份用于测试，在每个基本模型预测完所有样本之前，不会停止训练过程。此外，将原始特征与预测结果相结合，作为 stacking 模型第二层的输入，预测结果由第一层的多数投票得到。最后，将第二层的输入特征与输出结果结合起来作为最终特征，用于训练 GBDT 模型对钓鱼网页进行预测。表1总结了使用 URL 和 HTML 特性构建的钓鱼网页检测 stacking 模型的过程。



4 实验

4.1 数据集

（1）2K Phishing Detection Dataset (2K-PD)
  包含200个网页（URL+HTML），其中1000个合法网页（Alexa 排名100,000-101,000）以及1000个钓鱼网页（Phishtank 2017.7.12-2017.7.15）。
（2）50K Phishing Detection Dataset (50K-PD)
  包含49,947个网页（URL+HTML），其中30,873个合法网页（Alexa 排名10,000-12,000，其余28,873个从这些网页的超链接中收集）以及19,074个钓鱼网页（Phishtank 2009.6-2017.6）。图5和图6分别显示了2K-PD和50K-PD数据集中url长度的分布，可以观察到两个数据集url长度的分布是相似的。



（3）50K Image Phishing Detection Dataset (50K-IPD)
  包含53,103个网页（URL+HTML+Screenshots），其中28,320个合法网页（Alexa 排名10,000-15,000，其余23,320个从这些网页的超链接中收集）以及24,789个钓鱼网页（Phishtank 2009.6-2017.2）。该数据集用于评估视觉特征的有效性，50K-IPD 数据集中url长度分布如图7所示。



4.2 Baselines

  Baselines 包括一些流行的和最近提出的钓鱼网页检测方法，具体如下：
（1）CANTINA [26]：使用TF-IDF算法从HTML文本内容中提取关键字，并通过搜索引擎对其进行搜索，以检测网络钓鱼网页。作者还讨论了一些可用于降低 false positive rate 的简单启发式方法。
（2）Lightweight Phish Detector (LPD) [50]：一个基于搜索引擎的钓鱼网页检测系统，提取url的域名和网页标题，并用搜索引擎进行搜索。此外，还设计了一个决策算法，根据搜索结果检测网络钓鱼网页。
（3）URL Character Statistical Learning Classifiers (UCSLC) [51]：通过使用双样本Kolmogorov Smirnov检验来关注字符分布，并用一组选定的启发式特征补充它们以增强鲁棒性。

[26] Y. Zhang, J. Hong, L. Cranor, Cantina: a content-based approach to detecting phishing web sites, in: Proceedings of the 16th International Conference on World Wide Web - WWW’07, 2007, pp. 639–648, http://dx.doi.org/10.1145/ 1242572.1242659.
[50] G. Varshney, M. Misra, P.K. Atrey, A phish detector using lightweight search features, Comput. Secur. 62 (2016) 213–228, http://dx.doi.org/10.1016/j.cose. 2016.08.003.
[51] R. Verma, K. Dyer, On the character of phishing URLs: accurate and robust statistical learning classifier, in: Proceedings of the 5th ACM Conference on Data and Application Security and Privacy - CODASPY’15, 2015, pp. 111–122, http://dx.doi.org/10.1145/2699026.2699115.

4.3 与 Baselines 模型比较

  CANTINA 和 LPD 依赖于搜索引擎，难以应用于大规模数据，因此将基线方法应用于 2K-PD 数据集与本文方法进行比较。与本文方法不同的是，CANTINA 和 LPD 不需要训练数据，因此选取 2K-PD 数据集中70%的样本作为训练集，其余样本作为测试集，结果如表 2 所示。



4.4 与单个机器学习模型比较

  为了证明 stacking 的有效性，将本文提出的 stacking 模型与支持向量机(SVM)、最近邻分类器(NN)、决策树(DT)、随机森林(RF)、梯度增强决策树(GBDT)、XGBoost (XGB)和LightGBM (LGB)等单个机器学习模型进行比较。其中，GBDT、XGB 和 LGB 是本文提出的 stacking 模型中采用的基本模型。对 50K-PD 数据集进行评估，该数据集中70%作为训练集，其余为测试集。这些方法的性能如表 3 所示，可以观察到，在这三个指标上，本文提出的 stacking 方法比其他单一模型表现更好。



4.5 HTML string embedding 维度的影响

  本文使用 Word2Vec 来学习向量形式的 HTML string embedding，因此评估了 HTML string embedding 的维数的影响。如图 8 所示，随着 HTML string embedding 维数的增加，准确率先上升后下降，当 embedding 维度为200维时性能最佳。总体而言，HTML string embedding 维度的影响较小，波动率不超过0.5%。



4.5 比较 HTML string embedding 与人工特征

  在图 9 中，展示了使用 50K-PD 数据集在 GBDT 分类器上使用 HTML string embedding 和人工设计的特征(即上文中提到的所有特征中，除 HTML string embedding 以外的所有特征)的性能，HTML string embedding 的维数设置为200。从图中可以看出，HTML string embedding 在accuracy rate、missing rate和 false alarm rate 方面都非常接近于人工设计的特性。值得注意的是，HTML string embedding 不需要任何网络钓鱼领域的知识，embedding 学习过程不依赖于任何监督或外部信息。实验结果表明，HTML string embedding 对网络钓鱼网页检测具有重要意义。



4.6 特征的重要性

  利用 GBDT 分类器，使用特征的重要性系数来可视化特征的重要性，如图 10 所示，排名前3位的重要特征是‘‘HTML string embedding’’, ‘‘URL length information’’ 和 ‘‘Internal and external resources’’。有趣的是，HTML string embedding 的重要性非常高，甚至是排名第二的 URL length information 的10倍。



  此外，还评估了使用单一特征时的性能，结果如图 11 所示，其中特征按 accuracy 排序。其中，‘‘Alarm window’’, ‘‘Redirection’’, ‘‘Https’’, ‘‘Number ofdots in domain name’’ 和 ‘‘IP address’’单独使用时丢失率较高(接近100%)，说明它们是无效的。



  为了阐明本文提出的新特征的有效性，将这些特征进行划分：现有特征集 E，包括6个与url相关的特征和6个与 html 相关的特征；
new URL-related 特征集 U，包括2个特征；new HTML-related特征集 H，包括6个特征。然后比较了本文提出的 stacking 模型在不同特征集组合下的性能，如图 12 所示。实验结果表明，本文设计的新URL和HTML特征是有效的。



4.7 Stacking 模型中层数的影响

  如图 13 所示，当 stacking 层数设为2时，本文提出的 stacking 模型性能最佳。之后，随着层数的增加，stacking 模型的性能不会有明显的提高，这是由特性中包含的信息量所决定的。随着层数的增加，高层产出之间的差异越来越小，使得预测模型的收益越来越少。



4.8 基本模型选择说明

  一般来说，集成方法在使用多样化和有效的基本模型时效果更好。
  一方面，为了测量模型的多样性，使用 Kappa-Statistic [53] 来测量分类器对的预测结果。更具体地，给定两个分类器 $h_i$ 和 $h_j$：

  • a — 被 $h_i$ 和 $h_j$ 都预测为钓鱼网页的样本的数量；
  • b — 被 $h_i$ 预测为钓鱼网页且被 $h_j$ 预测为合法网页的样本的数量；
  • c — 被 $h_i$ 预测为合法网页且被 $h_j$ 预测为钓鱼网页的样本的数量；
  • d — 被 $h_i$ 和 $h_j$ 都预测为合法网页的样本的数量。
    然后可以根据下式估算 $h_i$ 和 $h_j$ 之间的 Kappa-measure：
    $K_p=\frac{p_1-p_2}{1-p_2}$
    $p_1=\frac{a+d}{a+b+c+d}$
    $p_2=\frac{(a+b)(a+c)+(c+d)(b+d)}{(a+b+c+d)^2}$
$k_p$ 越接近1，$h_i$ 和 $h_j$ 之间的差异就越小。

  另一方面，为了衡量模型的有效性，采用平均误差作为性能指标。选择支持向量机(SVM)、最近邻分类器(NN)、决策树(DT)、梯度增强决策树(GBDT)、XGBoost (XGB)和LightGBM (LGB)作为 stacking 的基本模型候选，采用5倍交叉验证对 50K-PD 数据集的训练集进行实验。如表 4 所示，一些分类器对的 $k_p$ 值较低，但平均误差较高。



  每个候选对的平均误差和 Kappa-Statistic 如图 14 所示。根据 low average error、high diversity 的原则，选择 GBDT、XGB 和 LGB 作为基本模型。



[53] T.G.D. Dragos D Margineantu, Pruning adaptive boosting, in: Proceedings of the 14th International Conference on Machine Learning, 1997, pp. 112–122.

4.8 实现细节

  Stacking 模型的参数包括两个方面，即每个基本模型的参数，以及 stacking 中的参数(如每个 stacking 层的基本模型个数，层数)。在训练阶段，训练集进行5倍交叉验证。每当调整一个参数时，固定其他参数来训练模型，并通过观察验证集上模型的性能来确定参数的值。
  逐层训练 stacking 模型，在各层的训练阶段对基本模型进行并行化训练。在对各基本模型进行训练的同时，对训练误差和评估误差进行验证。在训练时间成本方面，本文方法在 50K-PD 数据集(约35K训练样本)的训练集上花费64分钟(CPU: i7-6700HQ, RAM: 16G)。在模型的更新方面，当出现新的钓鱼数据时，采用增量学习策略对 XGB 和 LGB 进行训练，避免了从零开始训练；对于基本模型 GBDT，可以在训练集中加入新的钓鱼数据来训练以更新模型。

4.9 视觉特征探索

  使用 50K-IPD 数据集。设计了两种提取视觉特征的方法：
（1）使用一个预先训练的 ResNet18 模型进行特征提取，将ResNet18的输出修改为二维，并修正了CONV层的所有参数。在50K-IPD数据集上对模型进行训练，对网页图像进行分类。最后，以模型的倒数第二层的输出作为图像特征；
（2）设计了一个小型的卷积神经网络(CNN)，它有三个卷积层和三个全连接的层。在50K-IPD数据集上训练CNN模型，并以CNN的倒数第二层的输出作为网页的图像特征。
  表 5 为模型在有和没有视觉特性的情况下的性能比较，从表中可以看出，本文模型使用视觉特征后性能得到了显著的提高。当使用CNN提取的视觉特征时，性能得到了更明显的提高。然而，从 ResNet18 中提取的视觉特征贡献不大。原因可能是 ResNet18 的大部分参数都是在 ImageNet 上训练的。ImageNet 中的图片和网页上的图片之间有一个 gap。



5 未来工作

（1）数据集增强
（2）增强 HTML strings embedding 模型的鲁棒性
  要学习 HTML strings embedding，必须事先获得HTML字符串。缺点是它无法为从未出现在训练集或训练语料库中的新的 HTML string 学习 embedding。因此，必须考虑如何生成 HTML string 语料库，包括提取tags、variables 和 parameters作为语料库，或者在HTML代码中分割每个字符来创建语料库。
（3）目标识别
  研究人员已经开发了相当多的方法来检测钓鱼网页。然而，它们中的大多数通常依赖于多页面信息或搜索引擎，这是非常耗时的。计划设计一个基于单网页信息（特别是视觉特征），更高效地识别钓鱼目标的智能系统。
