Jail-Phish: An improved search engine based phishing detection system（Computers Security 2019）

发表于 2019-08-09 | 分类于 CCF-B |
字数统计: 3.7k 字 | 阅读时长 ≈ 14 分钟

1 作者介绍

一作：Routhu Srinivasa Rao（印度NITK Surathkal计算机科学与工程系）
Google Scholar：https://scholar.google.co.in/citations?user=1X_WI3UAAAAJ&hl=en



2 概述

  基于搜索引擎的技术的局限性在于，它们无法检测托管在受攻击服务器上的钓鱼网站。本文提出了一个名为 Jail-Phish 的应用程序，它可以提高基于搜索引擎的技术的准确性，能够检测受攻击服务器上托管的网络钓鱼站点（Phishing Sites Hosted on Compromised Servers, PSHCS）以及检测新注册的合法站点。Jail-Phish 比较可疑站点和搜索结果中匹配的域名，以计算它们之间的相似性得分。在同一网站的页面内存在某种程度的相似性，例如logo, favicon, images, scripts, styles 和 anchorlinks；而另一方面，PSHCS 中页面内的不相似性非常高。因此，使用可疑站点和匹配的域名之间的相似性得分作为检测 PSHCS 的参数。实验结果表明，Jail-Phish 的 accuracy 为98.61%，真阳性率为97.77%，假阳性率小于0.64%。

3 介绍

3.1 钓鱼网站分类

  攻击者以各种方式设计钓鱼网站，分为以下几种类别：

  • C1：对内容进行操纵的钓鱼网站，包括删除标题，版权，元数据等中的品牌名称，以及使用 NULL 或其本地链接替换合法超链接。
  • C2：网络钓鱼网站用单个图像替换整个合法文本。
  • C3：使用 URL 模糊技术的钓鱼网站（在商标名中拼写错误，或在子域或路径域中使用过多的点或冗长的 URL 保存商标名）。
  • C4：使用 HTTPS 连接的模仿高合法行为的钓鱼网站。
  • C5：免费网站托管服务器上托管的钓鱼网站。
  • C6：托管在受攻击 Web 服务器上的钓鱼网站。
  • C7：在 iframes 或 flash 中嵌入合法内容的钓鱼网站。


  图2显示了通过恶意注册和受攻击域名进行钓鱼攻击的数量。从图中可以看出，随着多年来钓鱼攻击的增加，使用受攻击服务器的钓鱼网站数量也在增加。与恶意注册相比，在受攻击的服务器上托管钓鱼网站的优点在于所设计的钓鱼网站也具有与受攻击域名相同的声誉，合法站点的声誉使得钓鱼网页可以绕过基于列表的技术和基于第三方服务的检测技术。

3.2 本文贡献

  本文通过动态搜索查询来解决以下基于搜索引擎方法的缺口：
（1）当钓鱼网站中的文本被替换为图像时，使用从版权、纯文本、标题文本中提取的标识关键字用于基于搜索引擎的技术无法提取相关术语。
（2）许多基于搜索引擎的技术难以解决托管在合法 web 服务器上的钓鱼网站，这是因为这些合法域名已经运行了很长时间，有更多的机会被搜索引擎索引。
（3）当使用域名+标题或网站标识关键字查询时，一些良性网站不会显示在搜索结果中，因而现有技术将合法网页分类为钓鱼网页。

  本文贡献：
（1）提出了一种动态搜索查询字符串的生成方法，用于返回相关搜索结果；
（2）提出了一种基于相似性的机制来检测托管在受攻击服务器(PSHCS)上的钓鱼网站；
（3）该技术对不同的 URL 伪装技术具有较强的鲁棒性，并能检测到 zero-day 网络钓鱼攻击；
（4）提出的方法能检测出非英文钓鱼网站，而且不依赖于事先访问目标网站资源数据库或网站历史；
（5）通过对新老网络钓鱼数据的综合评价表明，Jail-Phish 适用于long-lived 钓鱼站点的检测。

4 本文方法

  本文提出了一种有效的基于搜索引擎的技术，该技术对应于不同类别的网站有不同的搜索查询字符串，包括新的和不受欢迎的合法站点。本文还使用基于相似性的特征来检测受攻击服务器上的钓鱼网页。

4.1 domain + title 搜索查询中的观察

（1）In legitimate sites

  • 当查询域名和标题时，一些非英语站点不会显示在搜索结果中，但是当只查询域名时则会出现；
  • TLD 问题：当查询具有 ccTLD(country code top level domain)的域名时，搜索结果中显示具有 gTLD(global top level domain)的域名；
  • 如果 brand 名中没有标题，或者标题中的关键词更具竞争力，那么查询的域名就不会出现在搜索结果中。

（2）In phishing sites

  • 当受攻击的服务器上托管的钓鱼网站（PSHCS）中不存在标题时，搜索结果中会显示 query-domain。
  • 当 PSHCS 的标题中具有常规关键字(sign in、log in、login、home、untitled)时，搜索结果中显示 query-domain。

4.2 Jail-Phish

  工具采用 Java 实现，以 URL 为输入，从网站中提取标题、域名和源代码。提取的标题和域名用以生成搜索查询，考虑前 n 个搜索结果，如果查询域名出现在搜索结果中，则计算查询URL内容与SERP中匹配域名之间的相似性。如果相似性超过阈值，则将其分类为合法的，否则为PSHCS。另一种可能性是搜索结果中没有查询域名，最终被归类为钓鱼网站。这一假设也导致了将新注册的或不受欢迎的合法网站错误分类为网络钓鱼，但由于使用动态搜索查询，假阳性率(对网络钓鱼合法)大大降低。
  Jail-Phish的目标如下：（1）独立于语言；（2）高效的查询；（3）检测受攻击的域名；（4）基于客户端。



  将 Jail-Phish 分为6个阶段，伪代码如 Algorithm 1 所示：
（1）提取 domain 和 title
  使用 Selenium WebDriver 访问给定的 suspicious URL 并提取 source code，使用 Jsoup 库解析source code，提取 title 和其他相关的 DOM 文件。使用谷歌的 Guava 库从 URL 中提取 domain。
（2）Search query string 准备
  该模块为不同类型的网站提供动态搜索查询字符串。本文选择 domain + title 作为 search query，一般来说，domain 会给出相应网站的品牌名称，而 title 则会向在线用户和搜索引擎简要介绍网站。
  总体而言，本文的结论是：标题对于一些合法站点集的搜索查询是必须的，对于其他合法站点集应该跳过标题。因此，search query 将为 domain + title 或 domain。search query 初始化为 domain + title，在两种情况下改变：（1）返回搜索结果中的域名与 query domain 没有匹配的，则将 search query 更改为仅query domain；（2）给定的 URL 的源码中没有 title，search query 也将更改为仅 query domain。
（3）搜索处理
  使用 Google 作为搜索引擎，通过实验发现，搜索结果阈值的最优值 T=10。
（4）初始决策
  如果在前10个搜索结果中返回 query domain，则将domain status 更改为 true，这表明该 query domain 要么是纯合法页面，要么是 PSHCS。为了进一步确定，将匹配域的 baseURL 与查询域的 baseURL 进行比较。如果匹配，则将 URL 的查询域分类为合法域；否则将进行下一步筛选，在 matched domain 和给定的可疑页面之间执行相似性计算以区分合法页面和 PSHCS。

baseURL is termed as protocol + hostname + pathname of given URL

  存在一些查询域与搜索结果中的主域匹配，但在国家代码顶级域(ccTLD)或全局顶级域(gTLD)上有所不同，这类网站也经过相似度计算阶段。
  如果查询域与返回结果的任何域都不匹配，则仅使用 domain 重置查询字符串以执行搜索处理。 若还没有匹配，则将其归类为网络钓鱼，否则匹配的域 URL 将经历进一步的过滤阶段。
（5）相似性计算
  此模块检查给定的URL是否是受攻击服务器上托管的网络钓鱼页面。通常，网站内的页面之间存在一定程度的相似性，该假设用于检测 PSHCS。存在一些资源，如 Javascript，CSS，images，网站内所有页面共有的 logos。因此，从可疑网站 S 中提取这些资源文件，并在搜索结果中匹配域 M 以计算它们之间的相似性。算法2给出了相似度计算机制，基于相似度的特征如下：

  • URL：从它们各自的源代码中提取所有与S和M对应的本地URL，使用[href]从检查代码中提取本地锚链接，并将其保存在每个页面的不同集合中。
  • Styles：使用link[href]从检查代码中提取了M和S对应的所有局部样式，这些样式文件在链接URL中包含CSS扩展名。
  • Javascript：提取嵌入在这两个页面中的所有javascript文件，并将其存储在不同的集合中。这些文件都有js扩展名，是从script[src]中提取出来的。
  • Images：对img标签的src属性即img[src]进行检查，提取出与两个页面对应的所有图像。

  一旦从 JS、CSS、images、anchorlinks 中提取出所有这些资源url，它们就组合成两个大集合，对应于 M 和 S。将这两个集合作为输入馈送到 Jaccardian 相似性模块中，计算页面之间的相似性。



（6）最终决策
  如果相似度得分高于阈值，则将其分类为合法。为简单起见，本文将阈值设为0，也就是说，如果相似性得分高于0，表明存在某种程度的相似性，则将其归类为合法的，否则将其归类为托管在受攻击服务器上的钓鱼网站。

5.2 评价标准

  • phishing —— positive
  • legitimate —— negative
  • 正确预测的钓鱼网站 —— true positive (TP)
  • 正确预测的合法网站 —— true negative (TN)
  • 错误预测的合法网站 —— false positive (FP)
    （将合法网站错误的预测成了钓鱼网站）
  • 错误预测的钓鱼网站——false negative (FN)
    （将钓鱼网站错误的预测成了合法网站）
对于一个好的系统，FP应该是最小的，TP应该是最大的。

（1）True Positive Rate (TPR)
  在所有钓鱼网站中，正确预测的钓鱼网站(TP)的百分比，也被称为 Sensitivity 或 Recall。
  $TPR=\frac{TP}{TP+FN}100$
（2）True Negative Rate (TNR)
  在所有合法网站中，正确预测的合法网站(TN)的百分比，也被称为 Specificity。
  $TNR=\frac{TN}{TN+FP}100$
（3）False Positive Rate(FPR)
  在所有合法网站中，错误预测的合法网站(FP)的百分比。
  $FPR=\frac{FP}{FP+TN}100=1-Specificity$
（4）False Negative Rate(FNR)
  在所有钓鱼网站中，错误预测的钓鱼网站(FN)的百分比。
  $FNR=\frac{FN}{FN+TP}100=1-Recall$
（5）Accuracy (Acc)
  正确预测的合法网站和钓鱼网站占网站总数的百分比。
  $Acc=\frac{TP+TN}{TP+FP+TN+FN}100=\frac{TP+TN}{P+L}100$
（6）Precision (Pre)
  在所有预测的钓鱼网站中，正确预测的钓鱼网站的百分比。
  $Pre=\frac{TP}{TP+FP}100$
（7）F-Measure
  F-Measure 是 precision 和 recall 的调和平均值。F-Measure 总是更接近于 precision 和 recall 中的较小值，是一个评估 precision 和 recall 的综合度量。
  $F=2\frac{Pre*Recall}{Pre+Recall}$
