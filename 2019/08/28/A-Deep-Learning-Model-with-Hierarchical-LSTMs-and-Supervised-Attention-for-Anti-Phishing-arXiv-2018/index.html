A Deep Learning Model with Hierarchical LSTMs and Supervised Attention for Anti-Phishing（arXiv 2018）

发表于 2019-08-28 | 分类于 CCF-N |
字数统计: 4k 字 | 阅读时长 ≈ 15 分钟

1 作者介绍

一作：Minh Nguyen（越南河内科技大学）

2 概述

  本文研究了 NLP 文本分类的最新技术，以解决电子邮件的反网络钓鱼问题（即，预测电子邮件是否是网络钓鱼）。提出了一个框架，其中包含分层的长短时记忆网络（H-LSTM）和注意力机制，以便在单词和句子级别同时对电子邮件进行建模。
  在 H-LSTM 模型中，电子邮件被认为是一种层次化的体系结构，低层为单词(单词级)，高层为句子(句子级)。LSTM 模型首先在 word 级实现，其结果将传递给句子级的 LSTM 模型，从而生成整个电子邮件的表示向量。两个层次的 LSTM 模型的输出使用注意力机制进行组合，为电子邮件中的单词和句子分配贡献权重。如果电子邮件的标题可用，还集成了一个标题网络来对其进行建模。此外，还提出了一种基于单词在词汇表中的出现顺序，在分层 LSTMs 的单词级监督注意力机制的新技术。

2 本文模型

  钓鱼邮件检测是一个二进制分类问题，其形式化描述如下。
  设 $e={b, s}$ 为电子邮件，其中 $b$ 和 $s$ 分别为电子邮件的正文内容和标题。设 y 为二进制变量，表示 e 是否是钓鱼邮件。为了预测邮件的合法性，我们的目标是估计概率 $P(y=1|e)=P(y=1|b,s)$。接下来将描述分别使用 body 网络和 header 网络对 body $b$ 和 header $s$ 进行建模的方法，以实现此目标。

2.1 Body Network with Hierarchical LSTMs

  对于 body $b$，把它看作是 $b=(u_1, u_2, \dots, u_L)$，其中 $u_i$ 是第 $i$ 个句子，$L$ 是电子邮件正文 $b$ 中句子的数量。每个句子 $u_i$ 依次是 words/tokens 序列 $u_i=(v_{i, 1}, v_{i, 2}, \dots, v_{i, K})$，其中 $v_{i, j}$ 为句子 $u_i$ 中的第 $j$ 个 token，$K$ 为句子的长度。注意，将 $L$ 和 $K$ 设置为固定值，方法是用虚拟符号填充句子 $u_i$ 和 body $b$。
  由于 $b$ 中存在两个级别的信息（即，单词级别的单词 $v_{i, j}$，句子级别的句子 $u_i$），考虑一个包含两层双向 LSTMs 的分层网络来对这些信息进行建模。 具体地，第一层通过 embedding 模块，Bi-LSTM 模块和 attention 模块消费句子中的单词，以获得 $b$ 中的每个句子 $u_i$ 的表示向量（the word level layer）。 之后，第二层将来自第一层的表示向量与另一个 Bi-LSTM 和 attention 模块组合，得到整个电子邮件正文 $b$ 的表示向量（the sentence level layer）。然后将该表示向量用作估计 $P(y|b, s)$ 的特征并对初始电子邮件 $e$ 进行预测。

2.1.1 The Word Level Layer

（1）Embedding
  在 word level 层中，$b$ 中的每个句子 $u_i$ 中的每个单词 $v_{i, j}$ 首先转化为 embedding 向量 $w_{i, j}$，本文从一个大型语料库中预先训练好的 word embedding 矩阵 $W_e$ 中取对应的列向量 $w_{i, j}=W_e[v_{i, j}]$ （矩阵中每一列对应词汇表中的一个单词），得到 $w_{i, j}$。然后，$b$ 中的每个句子 $u_i=(v_{i, 1}, v_{i, 2}, \dots, v_{i, K})$ 将被转换成向量序列 $(w_{i, 1}, w_{i, 2}, \dots, w_{i, K})$，构成下一步 Bi-LSTM 模型的输入。

（2）Bidirectional LSTMs for the word level
  该模块使用正向和反向的两个 LSTMs 运行在每个输入向量序列 $(w_{i, 1}, w_{i, 2}, \dots, w_{i, K})$，前向 LSTM 生成前向隐向量序列 $(\vec{h_{i, 1}}, \vec{h_{i, 2}}, \dots, \vec{h_{i, K}})$，而向后 LSTM 产生向后隐向量序列 $(\overleftarrow{h_{i, 1}}, \overleftarrow{h_{i, 2}}, \dots, \overleftarrow{h_{i, K}})$。然后将这两个隐向量序列在每个位置连接起来，得到新的隐藏向量序列 $(h_{i, 1}, h_{i, 2}, \dots, h_{i, K})$ 用于 $b$ 中的句子 $u_i$，其中 $h_{i, j}=[\vec{h_{i, j}}, \overleftarrow{h_{i, j}}]$。隐藏向量 $h_{i, j}$ 的显著特征是，由于前向和后向 LSTMs 的重复属性，它对整个句子 $u_i$ 的上下文信息进行编码，尽管当前的单词 $v_{i, j}$ 受到了更大的关注。

2.3 Supervised Attention

  body 和 header 网络中的注意机制，期望对为信息丰富的单词/句子分配高权重，对邮件中不相关的词/句降级，以便于钓鱼检测。然而，只有提供大量的训练数据集来训练模型，才能实现这种理想的操作。在本文的钓鱼邮件检测案例中，训练数据集不够大，无法充分利用 attention 的优势。因此寻找有用的启发式问题，并将它们注入模型，以方便 attention 机制的运作。特别是，会先启发式地为句子中的每个单词确定一个分数，这样得分高的单词会被认为比得分低的单词对钓鱼检测更重要。之后，这些模型将被鼓励为接近启发式 importance scores 的单词生成注意力权重。我们的期望是，这种机制将有助于将我们的直觉引入注意力权重，以补偿训练数据集的小范围，从而潜在地提高模型的性能。假设句子中单词 $(v_{i, 1}, v_{i, 2}, \dots, v_{i, K})$ 的重要性得分分别为 $(g_{i, 1}, g_{i, 2}, \dots, g_{i, K})$，我们强制 attention 权重 $(\alpha_{i, 1}, \alpha_{i, 2}, \dots, \alpha_{i, K})$（等式1）接近重要性得分，通过惩罚在 attention 权重和重要性得分之间产生大的方差的模型。这相当于将方差加入等式 6 中的目标函数：

Importance scores 计算。为了计算重要性得分，我们的直觉是，如果一个单词在钓鱼邮件中频繁出现，而在合法邮件中出现的频率较低，那么这个单词对于钓鱼检测是很重要的。一个重要的单词不会出现在许多合法的电子邮件中，这一事实有助于消除大多数文档中使用的常见单词。因此，在我们的方法中，针对钓鱼电子邮件的频繁词汇会收到更高的影响分数。需要注意的是，我们查找钓鱼邮件中重要单词的方法与之前的工作不同，之前的工作只考虑钓鱼邮件中最频繁的单词，而忽略了它们在合法邮件中的出现。

3 评估

3.1 数据集

  本文中的模型是为了参与 IWSPA-AP 2018 而开发的。组织者提供了两个数据集来训练电子邮件钓鱼识别模型，第一个数据集包含只有 body 部分的电子邮件（称为 data-no-header），而第二个数据集包含 bodies 和 headers 的电子邮件（称为 data-full-header）。这两个数据集转换为两个共享任务，由参与者解决。这两个数据集的训练数据统计如表 2 所示。



3.2 预处理

  电子邮件正文中的大部分文本是非结构化的，这些句子通常很短，并且没有被句子的结尾符号（{. ! ?}）清楚的分开。为了将电子邮件主体分割成句子，本文开发了一个专门为数据集设计的内部句子分割器，通过考虑新行的第一个单词是否大写来确定句子的开头，或者一个大写的单词后面紧跟一个结束句符号。长度小于 3 的句子被组合成一个更长的句子，这大大减少了句子的数量，并在模型处理句子时扩展了句子中单词的上下文。图 3 显示了来自数据集的钓鱼电子邮件。



3.3 Baselines

  将提出的模型与基于 SVM 的基线模型进行比较。使用词汇表中单词的 tf-idf 评分作为基线的特征，采用了来自 sklearn 库的线性和非线性 SVM 的实现，其中通过 gensim toolkit 获得电子邮件的 tf-idf 表示。

3.4 Hyper-parameter Selection

  由于提供的数据集的大小很小，因此对提供的训练集使用 5 倍分层交叉验证，以搜索模型的最佳超参数，超参如下。
  在 body 和 header 网络中，对于所有 LSTMs，word embedding vector 的大小为300，cell 大小为 60。在 body 和 header 网络的 attention 模型上，attention vectors 的大小也设为 60。supervised attention 的 $\lambda$ 系数设置为 0.1，gradient clipping 的阈值是 0.3，drop-out 的 drop 率是 0.5。对于 Adam 更新规则，使用 0.0025 的学习率。最后，将线性 SVM baseline 设为 $C=10.0$，使用的 SVM 的非线性版本是带径向基函数内核的 C-SVC，其中 $(C,\gamma)=(50.0,0.1)$。
