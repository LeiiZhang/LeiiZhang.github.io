Leverage website favicon to detect phishing websites（SCN 2018）

发表于 2019-08-15 | 分类于 CCF-C |
字数统计: 5.7k 字 | 阅读时长 ≈ 21 分钟

1 作者介绍

一作：Kang Leng Chiew
研究所：马来西亚沙捞越大学计算机科学与信息技术学院
主页：http://www.archive.unimas.my/faculties/fcsit/chiew-kang-leng.html
Google Scholar：https://scholar.google.com.my/citations?user=fbeB3JEAAAAJ&hl=en



2 概述

2.1 摘要

  本文是以前的工作的一个扩展，利用 favicon 和谷歌图像搜索来揭示网站的身份。身份检索技术包括一种有效的数学模型，可用于帮助从搜索结果的许多条目中检索正确的身份。本文介绍了基于 favicon 的钓鱼攻击检测的增强版本，引入了域名扩展特征（Domain Name Amplification feature）并添加了附加特征（addition features）。当被检查的网站没有 favicon 时，附加特征非常有用。

选择 favicon 是因为它代表了网站的品牌，此外，favicon 不会受到网页上显示的动态内容(如广告)的影响。

2.2 本文贡献

（1）利用从网站中提取的 favicon，通过谷歌图像引擎发现潜在的钓鱼尝试。与当前的反钓鱼解决方案不同，本文提出的方法不需要对基于文本或基于图像的内容进行深入分析，提高了检测速度；
（2）数学方程的使用使得能够从搜索结果的许多条目中检索真实身份；
（3）提出了 additional features ，以克服缺失 favicon 的问题；
（4）本文方法不需要图像数据库或任何来自合法网站的预存信息，因此降低了由于数据库过时而导致高错误检测的风险。

A Keyword-based Combination Approach for Detecting Phishing Webpages（Computers & Security 2019）

发表于 2019-08-13 | 分类于 CCF-B |
字数统计: 5.9k 字 | 阅读时长 ≈ 22 分钟

1 作者介绍

一作：Yan Ding（新疆大学研究生）

2 概述

  文本提出了 Search & Heuristic Rule & Logistic Regression (SHLR) 相结合的方法，用以 a）检测钓鱼网站常用的混淆技术；b）提高过滤合法网页的效率。具体方法如下：
（1）基于搜索引擎识别合法网页
  将网页的 title 标签的内容作为搜索关键词输入到百度搜索引擎中，如果该网页的域名与前10名搜索结果中的任何一个域名匹配，则认为该网页是合法的;否则，将执行进一步的评估。

个人观点：匹配并不能直接判断为合法网页，也有可能是 PSHCS（Phishing Sites Hosted on Compromised Servers），Jail-Phish中的方法是匹配且baseURL（protocol+hostname+pathname）相同才判定为合法网页。

（2）基于启发式规则识别钓鱼网页
  如果网页不能被识别为合法的，则根据字符特征所定义的启发式规则，进一步检查网页是否为钓鱼网页。前两步可以快速过滤网页，满足实时检测的需要。
（3）基于 logistic 回归分类器识别网页
  利用 logistic 回归分类器对剩余页面进行评估，提高检测方法的适应性和准确性。

  实验结果表明，SHLR 可以过滤 61.9% 的合法网页，基于 URL 的词汇信息可识别 22.9% 的钓鱼网页。

A stacking model using URL and HTML features for phishing webpage detection（FGCS 2019）

1 作者介绍

一作：Yukun Li（广东工业大学自动化学院研究生）



2 概述

  本文设计了轻量级的 URL 和 HTML 特征，引入了 HTML string embedding。此外，还将 GBDT、XGBoost 和 LightGBM 多层结合，设计了一个 stacking 模型，使得不同的模型可以互补，从而提高了钓鱼网页检测的性能。本文收集了两个用于评估的真实数据集，分别为 50K-PD 和 50K-IPD。50K-PD 包含 49,947 个带有 URL 和 HTML代码的网页；50K-IPD 包含 53,103 个网页，除 URL 和 HTML 代码外还有 Screenshots。本文贡献如下：
（1）在特征提取方面，将人工设计的特征与 Word2Vec 模型提取的 HTML string embedding 相结合；
（2）结合几种机器学习模型的优点，提出了一种检测钓鱼网页的 stacking 模型；
（3）设计了一个实时的钓鱼检测系统，可以用来保护用户免受钓鱼攻击；
（4）发布了一个真实的钓鱼网页检测数据集。

Jail-Phish: An improved search engine based phishing detection system（Computers Security 2019）

发表于 2019-08-09 | 分类于 CCF-B |
字数统计: 3.7k 字 | 阅读时长 ≈ 14 分钟

1 作者介绍

一作：Routhu Srinivasa Rao（印度NITK Surathkal计算机科学与工程系）
Google Scholar：https://scholar.google.co.in/citations?user=1X_WI3UAAAAJ&hl=en



2 概述

  基于搜索引擎的技术的局限性在于，它们无法检测托管在受攻击服务器上的钓鱼网站。本文提出了一个名为 Jail-Phish 的应用程序，它可以提高基于搜索引擎的技术的准确性，能够检测受攻击服务器上托管的网络钓鱼站点（Phishing Sites Hosted on Compromised Servers, PSHCS）以及检测新注册的合法站点。Jail-Phish 比较可疑站点和搜索结果中匹配的域名，以计算它们之间的相似性得分。在同一网站的页面内存在某种程度的相似性，例如logo, favicon, images, scripts, styles 和 anchorlinks；而另一方面，PSHCS 中页面内的不相似性非常高。因此，使用可疑站点和匹配的域名之间的相似性得分作为检测 PSHCS 的参数。实验结果表明，Jail-Phish 的 accuracy 为98.61%，真阳性率为97.77%，假阳性率小于0.64%。

A new hybrid ensemble feature selection framework for machine learning-based phishing detection system（Information Sciences 2019）

主页：http://www.archive.unimas.my/faculties/fcsit/chiew-kang-leng.html
Google Scholar：https://scholar.google.com.my/citations?user=fbeB3JEAAAAJ&hl=en



2 概述

  本文为基于机器学习的钓鱼网站检测方法提出了一个新的特征选择框架，叫做Hybrid Ensemble Feature Selection (HEFS)。
  （1）在HEFS的第一阶段，提出了一种新的累积分布函数梯度(CDF-g)算法来生成主要特征子集，然后将这些特征子集输入到数据扰动集成中来生成次要特征子集。
  （2）第二阶段通过使用函数扰动集成从次要特征子集导出一组基线特征。
  （3）总体实验结果表明，当HEFS与随机森林分类器集成时表现最好，其中基线特征正确地区分了94.6％的网络钓鱼和合法网站，只使用了20.8％的原始特征。
  （4）另一个实验表明，在随机森林中使用基线特征(总共10个)优于在SVM、朴素贝叶斯、C4.5、JRip和部分分类器中使用所有特征(总共48个)。

Deep Learning for Phishing Detection （ISPA 2018）

发表于 2019-07-10 | 分类于 CCF-C |
字数统计: 2.9k 字 | 阅读时长 ≈ 11 分钟

1 作者介绍

一作： Wenbin Yao（PhD，北京邮电大学）

2 概述

  • 二维码钓鱼攻击：网络钓鱼攻击者改变写在其中的URL链接，并用一个精心设计的钓鱼链接替换它，但不改变 logo 图像，以欺骗用户和窃取用户信息。
  • 对于二维码攻击，通过检测嵌入在二维码中的 logo 来评估二维码中URL的合法性：logo识别 + 身份一致性 → 判断二维码的有效性（验证 logo 和域名之间的匹配来区分网站的合法性）。
  • 难点在于标志图像小，传统的目标识别方法降低了小目标的识别率。本文将改进的特征金字塔网络(FPN [3])与 Faster R-CNN 相结合，进行小尺度识别。
  • 测量了该方法在FlickrLogos-32 数据集上的效果。

[3] Lin, Tsung-Yi, “Feature Pyramid Networks for Object Detection.” CVPR. Vol. 1. No. 2. 2017.

选择 logo 进行检测的原因：（1）网站的 logo 比其他元素更有优势，因为它是官方和网站的代表；（2）网站 logo 和域名之间的关系是匹配的，logo 和域名之间的任何不匹配都将成为钓鱼攻击的标志。

Phishing URL Detection with Oversampling based on Text Generative Adversarial Networks（Big Data 2018）

字数统计: 3.3k 字 | 阅读时长 ≈ 13 分钟

1 作者介绍

一作：Ankesh Anand（博士）
研究所：加拿大蒙特利尔学习算法研究所
主页：https://mila.quebec/en/person/ankesh-anand/
Google Scholar：https://scholar.google.co.in/citations?user=HEozmkMAAAAJ&hl=en



2 概述

  在二分类任务中经常出现类的不平衡问题。如果一个类的数量超过另一个类，分类器将会严重偏向于数据量多的一类。可以对数据量少的类进行过采样，但现有方法在特征空间中进行过采样（将数据样本转化为特征空间中的向量，然后进行随机加权组合），删除了原始数据格式，用向量简洁地表示url。只有当特征定义正确且数据集多样化且不太稀疏时，这些方法才会有效。
  本文提出了一种数据空间中的过采样技术。使用少数类中的URL训练text-GANs，并生成可作为训练集一部分的合成URL。实验表明，采用过采样技术后，系统性能有了明显的提高。有趣的是，一些原始的测试url正是由text-GANs重新生成的。
